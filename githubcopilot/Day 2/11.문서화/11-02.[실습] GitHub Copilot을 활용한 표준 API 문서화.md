# 실습: GitHub Copilot을 활용한 표준 API 문서화

본 실습은 작성된 코드의 가독성을 높이는 'Google 스타일 주석'을 작성하고, 외부 개발자와 협업하기 위한 'OpenAPI(Swagger)' 명세를 자동으로 생성하는 과정을 다룹니다.

---

## 1. 실습용 기초 코드 (Base Code)

수강생은 아래의 간단한 상품 관리 API 코드를 바탕으로 문서화를 진행합니다.

**파일명: product_service.py**

```python
from typing import Optional, List, Dict

class ProductService:
    def __init__(self):
        self.products = [
            {"id": 1, "name": "Laptop", "price": 1200.0, "category": "Electronics"},
            {"id": 2, "name": "Coffee Maker", "price": 80.0, "category": "Home"}
        ]

    def get_product(self, product_id: int) -> Optional[Dict]:
        for product in self.products:
            if product['id'] == product_id:
                return product
        return None

    def add_product(self, name: str, price: float, category: str) -> Dict:
        new_id = len(self.products) + 1
        new_product = {"id": new_id, "name": name, "price": price, "category": category}
        self.products.append(new_product)
        return new_product
```

## 2. 실습 과제 (Tasks)

### Task 1: Google 스타일 Docstring 작성
* **미션:** `get_product`와 `add_product` 메서드에 Google Developer Documentation Style Guide를 준수하는 주석을 추가하세요.
* **요구사항:** 능동태(Active Voice)를 사용하고, `Args`와 `Returns` 섹션을 명확히 구분해야 합니다.
* **Copilot 프롬프트:** "이 함수에 대해 Google Style Guide를 준수하는 Docstring을 작성해줘. 능동태를 사용하고 인자와 반환값 설명을 상세히 포함해."

### Task 2: OpenAPI (Swagger) 명세 생성
* **미션:** 위 코드를 RESTful API로 노출한다고 가정하고, OpenAPI 3.0 표준에 맞는 YAML 명세서를 작성하세요.
* **요구사항:** 엔드포인트는 `/products/{id}` 및 `/products`로 설정하고, 200(성공)과 404(찾을 수 없음) 응답 구조를 포함해야 합니다.
* **Copilot 프롬프트:** "위 코드를 기반으로 OpenAPI 3.0 사양의 YAML 문서를 작성해줘. 각 엔드포인트의 파라미터와 응답 스키마를 포함해야 해."

### Task 3: 기술 문서 요약 (README)
* **미션:** 비전공자나 신입 개발자가 이 서비스의 역할을 이해할 수 있도록 마크다운 형식의 간단한 요약 문서를 작성하세요.
* **요구사항:** 구글 스타일 가이드의 '명확성'과 '간결함' 원칙을 지키며, 수동태 사용을 지양합니다.
* **Copilot 프롬프트:** "이 서비스의 기능을 비전공자도 이해할 수 있도록 간결한 README.md 형태로 요약해줘. 구글 스타일 가이드를 준수해서 작성해."

## 3. GitHub Copilot 활용 가이드

수강생은 문서화 작업 시 다음의 기법을 활용합니다.

1. **자동 완성 활용:** 주석의 시작 부분(`"""`)을 입력하고 엔터를 눌러 Copilot이 제안하는 주석 내용을 확인합니다.
2. **채팅창 활용:** 복잡한 API 구조는 Copilot Chat 창에 전체 코드를 전달하고 일괄적으로 YAML 변환을 요청합니다.
3. **용어 통일 요청:** "이 프로젝트에서 '상품'은 항상 Product로, '식별자'는 ID로 통일해서 문서를 작성해줘"와 같이 일관성을 강제하는 지시어를 사용합니다.
