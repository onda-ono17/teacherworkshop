# [최종 프로젝트] GitHub Copilot 실전 챌린지: 자가 진단 리스트

본 리스트는 프로젝트 완료 후 자신의 코드를 객관적으로 점검하고, AI와의 협업 과정을 복기하기 위한 도구입니다. 모든 항목에 체크할 수 있도록 최종 리팩토링을 진행해 주세요.

## 2. 주제별 심화 기술 (Topic Specific)

### 주제 A: 스마트 도서 대출 시스템 (API 중심)
* [ ] **동시성 제어:** 재고 차감 시 `@Lock(LockModeType.PESSIMISTIC_WRITE)`가 적용되었는가?
* [ ] **트랜잭션:** 대출 기록 생성과 재고 차감이 하나의 `@Transactional` 내에서 처리되는가?
* [ ] **경계값 테스트:** 재고가 0개일 때와 1개일 때의 시나리오가 단위 테스트에 포함되었는가?

### 주제 B: 실시간 날씨 데이터 분석 파이프라인 (데이터 중심)
* [ ] **장애 내성:** 외부 API 호출 시 `Retry` 또는 `Circuit Breaker` 설정이 포함되었는가?
* [ ] **데이터 가공:** Java Stream API를 활용하여 가독성 있게 데이터를 처리했는가?
* [ ] **예외 대응:** API 호출 실패 시 적절한 **Fallback 데이터**가 반환되는가?

## 개발자의 한 끗 차이 (Expert Tips)

> **"코드는 사람이 읽기 위해 작성되는 것이며, 기계는 그것을 실행할 뿐이다."**

* **의미 있는 이름:** `list1`, `data2` 대신 `overdueBooks`, `processedWeatherData`처럼 의도가 담긴 이름을 사용하세요. (Copilot에게 **"변수명을 클린 코드 원칙에 맞게 리팩토링해줘"**라고 요청해보세요.)
* **보안의 습관:** DB 접속 정보나 API Key가 `application.yml`에 평문으로 기록되어 있진 않나요? **환경 변수**를 활용하는 습관이 실무자의 실력입니다.

## 실습 종료 후 피드백 가이드

다음 3가지 질문을 스스로 던져 '설계 의도'를 확인해 보세요.

1. "AI가 제안한 코드 중에서 가장 마음에 들지 않아 **직접 수정한 부분**은 어디인가요?"
   - (의도: AI 결과물을 무비판적으로 수용하지 않았는지 확인)
2. "`.github/copilot-instructions.md` 파일이 코드를 생성할 때 실제로 어떤 변화를 주었나요?"
   - (의도: 지침 파일의 영향력과 프롬프트 엔지니어링의 효과 체감 확인)
3. "만약 데이터가 1,000만 건으로 늘어난다면, 지금 작성한 코드에서 어떤 부분이 **병목(Bottleneck)**이 될까요?"
   - (의도: 확장성과 성능 최적화에 대한 고려 여부 확인)
