# [최종 프로젝트] GitHub Copilot 실전 풀스택 챌린지

본 프로젝트는 2일간 학습한 **TDD, 리팩토링, 보안 코딩, 성능 최적화, 문서화** 기술을 하나의 완성된 서비스에 녹여내는 최종 관문입니다. GitHub Copilot과 협업하여 완성도 높은 결과물을 만들어내십시오.

---

## 1. 프로젝트 주제 (택 1 또는 아무거나)

수강생의 관심 분야에 맞춰 아래 시나리오 중 하나를 선택하여 진행합니다.

### 주제 A: AI 기반 스마트 도서 대출 시스템 (API & 로직 중심)
* **핵심 도전:** 복잡한 비즈니스 규칙(연체료 가산, 회원 등급별 대출 제한) 구현.
* **주요 기능:**
    * 대출 권수 제한: 일반(3권), VIP(7권), 관리자(무제한).
    * 스마트 연체료: 연체 1일당 500원 부과 (주말 및 공휴일 제외 로직 포함).
    * 예약 상태 관리: 대출 중인 도서 예약 및 반납 시 자동 상태 전환.

### 주제 B: 실시간 날씨 데이터 분석 파이프라인 (데이터 & I/O 중심)
* **핵심 도전:** 외부 API 연동, 결측치 정제 및 대량 데이터 처리 성능 최적화.
* **주요 기능:**
    * 외부 API 연동: 가상의 날씨 API로부터 JSON 데이터 수집 및 예외 처리(Retry 로직).
    * 데이터 분석: 특정 기간 내 최고/최저 기온 탐색 및 이상치(Outlier) 제거.
    * 성능 최적화: 수만 건의 데이터 처리 시 $O(n^2)$ 루프를 제거하고 최적화된 자료구조 사용.

## 2. 프로젝트 수행 단계 (60분 타임어택 워크플로우)

수강생들이 체계적으로 진행할 수 있도록 AI와의 협업 단계를 명확히 지키며 수행합니다.

### 1단계: 요구사항 분석 및 스캐폴딩 (15분)
* **미션:** 요구사항을 분석하여 프로젝트 구조와 의존성을 설정합니다.
* **Copilot 활용:** `@workspace /new` 명령어로 폴더 구조와 `requirements.txt` 또는 `pyproject.toml`을 생성합니다.
* **프롬프트 예시:** "스마트 도서 대출 시스템을 위한 Flask 기반 프로젝트 구조를 만들어줘. 필요한 폴더는 models, services, tests로 구분해."

### 2단계: 도메인 모델 및 핵심 로직 구현 (15분)
* **미션:** 클린 코드 원칙을 지키며 핵심 비즈니스 기능을 개발합니다.
* **Copilot 활용:** 함수의 시그니처와 주석을 먼저 작성하여 Ghost Text(회색 제안 코드)를 유도합니다.
* **프롬프트 예시:** "회원 등급과 현재 대출 권수를 확인하여 대출 가능 여부를 불리언 값으로 반환하는 check_eligibility 함수를 구현해줘."

### 3단계: 보안 감사 및 리팩토링 (10분)
* **미션:** OWASP Top 10 기준 취약점을 점검하고 코드 스멜을 제거합니다.
* **Copilot 활용:** 작성된 코드를 선택하고 보안 취약점 분석을 요청합니다.
* **프롬프트 예시:** "/refactor 이 코드에서 SQL Injection 위험이 있는 부분을 고쳐주고, 장황한 if-else 문을 가드 클로즈 패턴으로 개선해줘."

### 4단계: 단위 테스트 및 검증 (10분)
* **미션:** 핵심 로직의 안정성을 보장하기 위한 테스트 코드를 작성합니다.
* **Copilot 활용:** `/tests` 명령어를 사용하여 경계값 및 예외 케이스 테스트를 생성합니다.
* **프롬프트 예시:** "연체료 계산 함수에 대해 주말이 포함된 경우와 포함되지 않은 경우를 모두 검증하는 pytest 코드를 작성해줘."

### 5단계: 문서화 및 마무리 (10분)
* **미션:** API 명세서(Swagger)와 README.md를 완성합니다.
* **Copilot 활용:** Google Style Guide에 맞춰 Docstring을 생성하고 프로젝트 요약본을 작성합니다.
* **프롬프트 예시:** "이 프로젝트의 모든 기능을 분석해서 기술 스택과 실행 방법이 포함된 README.md를 구글 스타일 가이드에 맞춰 작성해줘."

## 3. 최종 성과 측정 체크리스트 (Rubric)

* [ ] **기능성:** 모든 요구사항이 정상적으로 동작하며 테스트를 통과하는가?
* [ ] **가독성:** 변수명이 의미 있고 메서드가 단일 책임을 갖도록 리팩토링되었는가?
* [ ] **보안성:** 하드코딩된 비밀번호나 주입 공격(Injection) 취약점이 없는가?
* [ ] **문서화:** 외부 개발자가 문서만 보고도 프로젝트를 즉시 실행할 수 있는가?
