# [실습 자료] 취약점이 포함된 코드 샘플 및 보안 실습 (VulnerableApp)

본 실습은 실제 보안 사고의 주범인 SQL Injection과 XSS 취약점을 포함한 코드를 분석하고, GitHub Copilot을 통해 이를 '시큐어 코딩(Secure Coding)'으로 전환하는 과정을 다룹니다.

---

## 1. 실습용 소스 코드

**파일명: vulnerable_app.py**

```python
import sqlite3
from flask import Flask, request, render_template_string

app = Flask(__name__)

# [취약점 1] SQL Injection: 사용자 입력을 쿼리문에 직접 연결
@app.route('/user_info')
def get_user_info():
    user_id = request.args.get('id')
    db = sqlite3.connect('users.db')
    cursor = db.cursor()
    # 공격 예시: ?id=1 OR 1=1
    query = f"SELECT * FROM users WHERE id = {user_id}"
    cursor.execute(query)
    user = cursor.fetchone()
    return str(user)

# [취약점 2] XSS (Cross-Site Scripting): 사용자 입력을 필터링 없이 렌더링
@app.route('/hello')
def hello():
    name = request.args.get('name', 'Guest')
    # 공격 예시: ?name=<script>alert('XSS')</script>
    template = f"<h1>Hello, {name}!</h1>"
    return render_template_string(template)
```

## 2. 단계별 실습 시나리오 (Hands-on)

### 1단계: 보안 감사 (Security Audit)
* **미션:** Copilot Chat에게 현재 코드의 보안 취약점을 찾아내고 어떤 종류의 공격에 취약한지 설명하게 합니다.
* **프롬프트:** "이 코드에서 OWASP Top 10에 해당하는 보안 취약점을 모두 찾아줘. 특히 SQL Injection과 XSS 위협이 있는지 분석해줘."
* **학습 포인트:** AI가 문자열 포매팅(`f-string`)을 사용한 쿼리문과 필터링 없는 HTML 렌더링의 위험성을 정확히 지적하는지 확인합니다.



### 2단계: 취약점 수정 (Remediation)
* **미션:** Copilot의 제안을 받아 안전한 코드로 수정합니다.
* **프롬프트 1 (SQL Injection):** "이 쿼리를 SQL Injection을 방지할 수 있는 Parameterized Query(Prepared Statement) 방식으로 수정해줘."
* **프롬프트 2 (XSS):** "사용자 입력을 안전하게 렌더링하기 위해 Jinja2 템플릿 엔진의 자동 이스케이프 기능을 사용하거나 필터링하는 코드로 바꿔줘."
* **학습 포인트:** 데이터와 코드를 분리하는 방식(Parameterized Query)과 특수 문자를 무해하게 변환하는 '이스케이프(Escape)'의 원리를 이해합니다.

### 3단계: 보안 제약 조건 추가 (Hardening)
* **미션:** 프롬프트에 구체적인 보안 요구사항을 명시하여 처음부터 견고한 코드를 생성하게 합니다.
* **프롬프트:** "너는 보안 전문가야. 사용자로부터 파일 이름을 입력받아 서버에 저장하는 기능을 만들려고 해. 파일 경로 조작(Path Traversal) 공격을 방지할 수 있는 시큐어 코딩 규칙을 적용해서 Python 코드를 작성해줘."
* **학습 포인트:** AI에게 '보안 전문가'라는 페르소나를 부여했을 때 생성되는 코드의 품질(입력값 검증, 경로 정규화 등) 차이를 경험합니다.
