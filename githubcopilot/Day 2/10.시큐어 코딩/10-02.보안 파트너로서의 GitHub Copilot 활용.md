# 보안 파트너로서의 GitHub Copilot 활용

개발 프로세스 초기 단계에서 보안 취약점을 발견하고 수정하는 'Shift-Left Security'를 구현하기 위해 GitHub Copilot을 보안 전문가처럼 활용하는 전략을 학습합니다.

---

### 1. 실시간 보안 스캐닝 (Real-time Security Scanning)

Copilot은 코드를 작성하는 순간에도 배경에서 보안 취약점 패턴을 감시합니다.

* **자동 감지 기능:** - 소스 코드에 **하드코딩된 API Key, 패스워드, 인증 토큰**이 포함될 경우 즉시 경고를 표시합니다.
    - SQL Injection이나 XSS(Cross-Site Scripting)를 유발할 수 있는 취약한 함수 호출 패턴을 인식하여 대안을 제시합니다.
* **학습 포인트:** 에디터 하단의 문제(Problems) 탭이나 코드상의 물결선(Squiggle)을 통해 AI가 제안하는 보안 경고를 즉각 확인하는 습관을 기릅니다.



---

### 2. 보안 리팩토링 (Security Refactoring)

취약점이 발견된 레거시 코드를 최신 보안 표준에 맞는 안전한 코드로 변환합니다.

* **Prepared Statement 전환:** 문자열 결합 방식의 SQL 쿼리를 발견하면, 자동으로 매개변수화된 쿼리(Prepared Statement)로 수정할 것을 제안합니다.
* **안전한 라이브러리 추천:** 구식의 취약한 암호화 알고리즘(예: MD5, SHA-1) 대신 강력한 최신 라이브러리(예: Argon2, BCrypt)를 사용하도록 코드를 재구성합니다.
* **실전 예시:**
    - **취약한 코드:** `query = "SELECT * FROM users WHERE id = '" + user_id + "'"`
    - **리팩토링 요청:** "이 쿼리를 SQL Injection 방지를 위해 Prepared Statement 방식으로 바꿔줘."
    - **결과:** `cursor.execute("SELECT * FROM users WHERE id = %s", (user_id,))`



---

### 3. 보안 전용 프롬프트 전략 (Security Prompting)

수동적인 경고를 넘어, 능동적으로 코드의 보안성을 심사(Audit)하도록 지시합니다.

* **OWASP 기준 심사:** > "이 파일의 코드를 OWASP Top 10 기준으로 분석해서 잠재적인 보안 취약점을 모두 찾아줘. 특히 권한 설정 오류(Broken Access Control)가 있는지 중점적으로 봐줘."
* **공격자 관점 시뮬레이션:** > "만약 네가 화이트 해커라면, 이 API 엔드포인트에서 어떤 방식으로 주입 공격(Injection)을 시도할 것 같아? 방어 코드도 함께 작성해줘."
* **에러 메시지 마스킹:** > "예외 처리 로직에서 시스템 내부 정보가 유출되지 않도록 에러 메시지를 사용자 친화적이고 안전하게 수정해줘."

---

### 4. 핵심 포인트

* **자동화된 방어선:** Copilot의 실시간 스캔 기능은 개발자의 실수를 줄여주는 훌륭한 '안전장치'임을 강조합니다.
* **비판적 수용:** AI가 보안상 안전하다고 판단하더라도, 조직의 보안 정책이나 최신 위협 동향에 비추어 사람이 최종 검토해야 합니다.
* **보안 교육 도구:** 취약점이 지적되었을 때 단순히 고치는 것에 그치지 않고, "왜 이 코드가 위험한지" Copilot에게 물어보며 보안 지식을 내재화하도록 유도합니다.
