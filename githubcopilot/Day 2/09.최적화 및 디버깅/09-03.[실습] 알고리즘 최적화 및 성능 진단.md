# [실습 시나리오] 알고리즘 최적화 및 성능 진단 (Hands-on)

본 실습은 비효율적인 $O(n^2)$ 알고리즘을 분석하고, GitHub Copilot의 제안을 바탕으로 $O(n)$ 수준으로 성능을 개선하는 과정을 다룹니다. 또한, 논리적 오류가 있는 코드를 AI와 함께 디버깅하는 기법을 학습합니다.

---

## 1. 실습 자료: 비효율적 알고리즘 샘플

수강생들은 아래의 비효율적인 중복 항목 찾기 코드를 기반으로 실습을 시작합니다.

**파일명: duplicate_finder.py**

```python
# 두 리스트에서 공통된 항목을 찾는 비효율적인 코드
def find_common_items(list1, list2):
    common = []
    # [문제점] O(n^2) - 중첩 루프 발생 및 내부 리스트 검색
    for item1 in list1:
        for item2 in list2:
            if item1 == item2:
                # [문제점] 'in common' 역시 리스트 검색이므로 O(k) 추가 발생
                if item1 not in common:
                    common.append(item1)
    return common

# 실행 예시: 데이터가 커질수록 기하급수적으로 느려짐
large_list1 = list(range(10000))
large_list2 = list(range(5000, 15000))

# 성능 측정을 위해 실행
import time
start = time.time()
result = find_common_items(large_list1, large_list2)
print(f"걸린 시간: {time.time() - start:.4f}초")
```

## 2. 단계별 실습 시나리오

### 1단계: 복잡도 분석 및 예측
* **미션:** Copilot Chat에게 현재 함수의 시간 복잡도를 물어보고, 데이터 규모에 따른 성능 병목 원인을 파악합니다.
* **프롬프트:** "이 함수의 시간 복잡도와 공간 복잡도를 분석해주고, 리스트 크기가 각각 10만 개로 커질 때 어떤 성능 문제가 생기는지 구체적으로 알려줘."
* **학습 포인트:** AI가 중첩 루프뿐만 아니라 'in common' 연산의 오버헤드까지 정확히 계산하는지 확인합니다.

### 2단계: 최적화 제안 및 구현
* **미션:** $O(n^2)$ 로직을 $O(n)$으로 개선하기 위해 Set 자료구조를 적용합니다.
* **프롬프트:** "이 코드를 Set(집합) 자료구조를 사용하여 최적화해줘. 시간 복잡도를 $O(n)$으로 줄여야 하고, 결과의 중복은 제거되어야 해."
* **학습 포인트:** 리스트의 선형 탐색($O(n)$)을 해시 기반 탐색($O(1)$)으로 교체했을 때의 극적인 속도 차이를 체감합니다.

### 3단계: 논리 디버깅 챌린지
* **미션:** 아래의 '오류가 있는 정렬 코드'를 제공받아 Copilot과 함께 수정합니다.

**챌린지용 코드 (버그 포함):**
```python
def buggy_bubble_sort(arr):
    n = len(arr)
    # [버그] 루프 범위 설정 오류 및 부등호 방향 오류
    for i in range(n):
        for j in range(0, n - i): 
            if arr[j] < arr[j + 1]: 
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr
```
* **활동:** 코드를 선택한 후 인라인 채팅(Ctrl+I) 또는 채팅창을 활용합니다.
* **프롬프트:** "/fix 이 코드의 정렬이 제대로 되지 않고 에러가 발생해. 원인을 찾아서 고쳐주고 왜 그런 에러가 났는지 설명해줘."

## 3. 핵심 포인트

* **성능의 시각화:** $O(n^2)$ 코드는 데이터가 10배 늘어나면 시간이 100배 늘어난다는 점을 실제 실행 시간 측정을 통해 입증하십시오.
* **적절한 자료구조 선택:** 상황에 맞는 자료구조(Set, Dictionary 등)를 선택하는 것이 성능 최적화의 핵심임을 강조하십시오.
* **AI 디버깅의 활용:** AI가 로직의 모순(인덱스 범위 등)을 찾아내는 과정을 보여주되, 수정된 결과가 의도한 방식(오름차순/내림차순)인지 반드시 확인하도록 지도하십시오.
