## 1. 실습용 레거시 코드 (Java)

수강생들이 프로젝트에 추가해야 할 기초 코드입니다. 주석이 없고 로직이 고도로 결합된 상태입니다.

### Item.java
```java
public class Item {
    public String name;
    public int sellIn;
    public int quality;

    public Item(String name, int sellIn, int quality) {
        this.name = name;
        this.sellIn = sellIn;
        this.quality = quality;
    }

    @Override
    public String toString() {
        return this.name + ", " + this.sellIn + ", " + this.quality;
    }
}
```

### GildedRose.java
```java
class GildedRose {
    Item[] items;

    public GildedRose(Item[] items) {
        this.items = items;
    }

    public void updateQuality() {
        for (int i = 0; i < items.length; i++) {
            if (!items[i].name.equals("Aged Brie")
                    && !items[i].name.equals("Backstage passes to a TAFKAL80ETC concert")) {
                if (items[i].quality > 0) {
                    if (!items[i].name.equals("Sulfuras, Hand of Ragnaros")) {
                        items[i].quality = items[i].quality - 1;
                    }
                }
            } else {
                if (items[i].quality < 50) {
                    items[i].quality = items[i].quality + 1;
                    if (items[i].name.equals("Backstage passes to a TAFKAL80ETC concert")) {
                        if (items[i].sellIn < 11) {
                            if (items[i].quality < 50) {
                                items[i].quality = items[i].quality + 1;
                            }
                        }
                        if (items[i].sellIn < 6) {
                            if (items[i].quality < 50) {
                                items[i].quality = items[i].quality + 1;
                            }
                        }
                    }
                }
            }
            if (!items[i].name.equals("Sulfuras, Hand of Ragnaros")) {
                items[i].sellIn = items[i].sellIn - 1;
            }
            if (items[i].sellIn < 0) {
                if (!items[i].name.equals("Aged Brie")) {
                    if (!items[i].name.equals("Backstage passes to a TAFKAL80ETC concert")) {
                        if (items[i].quality > 0) {
                            if (!items[i].name.equals("Sulfuras, Hand of Ragnaros")) {
                                items[i].quality = items[i].quality - 1;
                            }
                        }
                    } else {
                        items[i].quality = items[i].quality - items[i].quality;
                    }
                } else {
                    if (items[i].quality < 50) {
                        items[i].quality = items[i].quality + 1;
                    }
                }
            }
        }
    }
}
```

## 2. 실습 단계 및 프롬프트 가이드

### 1단계: AI를 활용한 로직 역설계 (Reverse Engineering)
복잡한 코드를 읽기 전, AI에게 요약 및 구조화를 요청하여 '인지적 부하'를 줄입니다.

* **프롬프트:** "이 `updateQuality` 메서드의 비즈니스 로직을 분석해줘. 특히 상품 이름(Aged Brie, Sulfuras, Backstage pass)에 따라 `quality`와 `sellIn`이 어떻게 변하는지 표로 정리해줘."
* **학습 포인트:** 인간이 조건문을 일일이 추적하는 대신, AI가 비즈니스 규칙을 명세서 수준으로 복원해주는 경험을 합니다.

### 2단계: 특성 테스트(Characterization Test) 작성
리팩토링의 대전제는 **"동작 보존"**입니다. 로직을 고치기 전 현재 상태를 고정하는 '안전 그물'을 만듭니다.

* **프롬프트:** "이 레거시 코드의 현재 동작을 보존하기 위한 JUnit 5 테스트 코드를 작성해줘. Aged Brie, Sulfuras, 일반 상품 각각에 대해 하루가 지났을 때의 결과를 검증하는 테스트여야 해."
* **학습 포인트:** Michael Feathers가 강조한 **"테스트 없는 리팩토링은 리팩토링이 아니다"**라는 원칙을 실천합니다.



### 3단계: 접합부(Seams) 식별 및 리팩토링 제안
수정이 용이한 지점을 찾고, 객체지향적 설계를 도입할 준비를 합니다.

* **프롬프트:** "이 코드에서 Michael Feathers가 말하는 '접합부(Seam)'를 찾는다면 어디가 될까? 그리고 이 중첩된 if문을 다형성(Polymorphism)이나 전략 패턴(Strategy Pattern)으로 바꿀 수 있는 구조를 제안해줘."
* **학습 포인트:** 복잡한 `if-else` 덩어리를 어떻게 독립적인 클래스로 분리할 수 있는지, AI의 아키텍처 설계 역량을 활용합니다.
