# 실습: 스파게티 코드 분석 및 리팩토링 (LegacyInvoiceProcessor)

본 자료는 실제 현업에서 마주칠 수 있는 전형적인 레거시 코드의 특징을 담고 있습니다. 가독성이 낮고 의존성이 강하게 결합된 코드를 분석하고, GitHub Copilot을 활용하여 안전하게 리팩토링하는 과정을 학습합니다.

---

## 1. 레거시 코드의 주요 특징 (Code Smells)

제공된 소스 코드는 다음과 같은 문제점을 가지고 있습니다.

* **전역 변수(global db) 사용:** 함수의 입력과 출력 외에 외부 상태에 의존하여 예측 불가능한 결과를 초래합니다.
* **마법의 숫자(Magic Numbers):** 'type 1', '0.9' 등 의미를 알 수 없는 숫자들이 로직에 직접 사용되었습니다.
* **책임의 혼재:** 할인 로직(비즈니스), 데이터 저장(DB), 알림 발송(I/O), 로그 출력(Print)이 하나의 함수에 섞여 있습니다.
* **외부 의존성 결합:** send_alert 함수가 내부에 직접 호출되어 있어, 실제 이메일 발송 없이 로직만 테스트하기가 불가능합니다.

## 2. 실습용 소스 코드

**파일명: legacy_processor.py**
```
import datetime

db = [] # 전역 변수 DB 시뮬레이션

def process_it(data):
    global db
    for i in range(len(data)):
        # 마법의 숫자와 복잡한 조건문
        if data[i]['type'] == 1:
            if data[i]['amt'] > 100000:
                t = data[i]['amt'] * 0.9
            else:
                t = data[i]['amt']
        elif data[i]['type'] == 2:
            t = data[i]['amt'] * 0.85
        
        # I/O 로직과 비즈니스 로직의 혼재
        print(f"Processing {data[i]['id']}...") 
        
        d = datetime.datetime.now().strftime("%Y-%m-%d")
        record = {'id': data[i]['id'], 'final_amt': t, 'date': d}
        
        # 중복 로직과 검증 부재
        db.append(record)
        if t > 50000:
            send_alert(data[i]['id']) # 외부 의존성 직접 결합
```

## 3. GitHub Copilot 활용 실습 단계

### 단계 1: 역설계 및 의도 파악
* **활동:** 코드를 전체 선택한 후 Copilot Chat에 질문하여 로직을 요약합니다.
* **프롬프트:** "이 함수는 비즈니스 관점에서 무엇을 하는 코드인지 요약해줘. 특히 할인 규칙과 데이터 저장 방식 위주로 설명해줘."

### 단계 2: 특성 테스트(Characterization Test) 구축
* **활동:** 코드를 수정하기 전, 현재의 동작을 그대로 기록하는 테스트를 만듭니다.
* **프롬프트:** "이 함수의 현재 동작을 보존하기 위한 특성 테스트(Characterization Test)를 pytest로 작성해줘. type 1과 2, 그리고 금액 경계값인 100,000원에 대한 케이스를 포함해줘."

### 단계 3: 접합부(Seams) 추출 및 리팩토링
* **활동:** 외부 의존성을 분리하고 전역 변수를 제거합니다.
* **프롬프트:** "/fix 이 코드에서 전역 변수 db를 제거하고 클래스 멤버나 인자로 변경해줘. 또한 send_alert를 인터페이스나 의존성 주입(DI) 방식으로 처리하여 테스트 가능하게 만들어줘."
