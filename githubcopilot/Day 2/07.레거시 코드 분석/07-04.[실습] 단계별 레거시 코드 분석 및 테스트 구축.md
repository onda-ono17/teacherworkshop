# 실습: 단계별 레거시 코드 분석 및 테스트 구축

본 실습은 앞서 제공된 `LegacyInvoiceProcessor` 코드를 대상으로 GitHub Copilot을 활용하여 분석부터 테스트 구축까지의 전 과정을 단계별로 수행합니다. Michael Feathers의 레거시 코드 개선 전략을 실무에 어떻게 적용하는지 체감하는 것이 핵심입니다.

---

### 1단계: 코드 설명 및 의도 파악 (/explain)

**미션:**
Copilot Chat에게 `/explain` 명령어를 사용하여 코드의 전체적인 흐름과 전역 변수의 역할을 설명하도록 요청합니다.

* **수행 방법:** `legacy_processor.py` 파일의 `process_it` 함수를 선택한 뒤 채팅창에 `/explain`을 입력합니다.
* **학습 포인트:**
    - AI가 복잡한 if-elif 조건문과 전역 변수 `db` 사이의 관계를 정확히 짚어내는지 확인합니다.
    - 코드에 명시되지 않은 '할인 로직'과 '데이터 영속화'라는 비즈니스 의도를 AI가 어떻게 해석하는지 관찰합니다.

---

### 2단계: 기술 부채 및 위험 요소 식별

**미션:**
Michael Feathers의 관점에서 코드를 비판적으로 바라보고, 개선을 위한 '접합부(Seams)'를 제안받습니다.

* **프롬프트 예시:** "이 코드에서 유지보수를 어렵게 만드는 기술 부채 3가지만 지적해줘. 그리고 Michael Feathers의 관점에서 의존성을 분리하기 위해 어떤 '접합부(Seam)'를 만들 수 있을지 제안해줘."
* **예상 결과:** - 전역 변수 의존성, `datetime.now()` 하드코딩, `send_alert`와의 강한 결합(Tight Coupling) 지적.
    - 객체 접합부(Object Seams)를 이용한 의존성 주입(DI) 제안 확인.

---

### 3단계: 특성 테스트(Characterization Test) 생성

**미션:**
리팩토링을 시작하기 전, 현재 코드의 결과값을 완벽하게 보존하기 위한 '안전 그물(Safety Net)'을 구축합니다.

* **프롬프트 예시:** "이 코드의 로직을 고치지 않고, 현재 입력에 따른 결과가 그대로 유지되는지 확인하기 위한 특성 테스트(Characterization Test)를 pytest로 작성해줘. 전역 변수 db와 외부 함수 send_alert는 unittest.mock을 사용해서 격리해줘."
* **학습 포인트:**
    - 리팩토링의 전제 조건은 '동작 보존'임을 이해합니다.
    - AI가 `MagicMock`을 활용하여 전역 변수와 외부 호출을 어떻게 가로채는지(Sensing) 확인합니다.
    - 작성된 테스트를 실행하여 현재 코드의 동작이 기록되는지 확인합니다.
