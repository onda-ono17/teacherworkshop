# Custom Instructions: 팀 코딩 컨벤션 자동화 (copilot-instructions.md)

이전 단원에서 배운 'Paradigm(패러다임)'과 'Specificity(구체성)' 프롬프트를 채팅할 때마다 매번 작성하는 것은 매우 번거롭습니다. 
GitHub Copilot은 **.github/copilot-instructions.md** 파일을 통해 프로젝트나 팀 단위의 전역 프롬프트(Global Prompt)를 설정하는 기능을 제공합니다.

---

## 1. Custom Instructions의 역할

* **배경 지식 주입:** 현재 프로젝트의 기술 스택, 아키텍처, 데이터베이스 제약 사항 등을 Copilot에게 사전에 가르칩니다.
* **코딩 스타일 강제:** 변수명 짓는 규칙(CamelCase vs Snake_case), 주석 작성 언어(영어 vs 한국어) 등을 지정합니다.
* **작동 원리:** 이 파일이 프로젝트 환경에 존재하면, 사용자가 Copilot Chat에 질문을 던질 때 **이 파일의 내용이 보이지 않는 배경 프롬프트로 자동으로 함께 전송**되어 답변의 품질과 형태를 고정시킵니다.

---

## 2. 파일 생성 및 우선순위

1. **프로젝트 레벨:** .github/copilot-instructions.md (가장 권장되며, 팀원 전체에게 동일하게 적용됨)
2. **VS Code 워크스페이스 레벨:** .vscode/copilot-instructions.md
3. 파일 내부에 일반적인 마크다운(Markdown) 문법으로 지시사항을 나열하면 됩니다.

---

## 3. 실무 작성 예시 (Best Practice)

`markdown
# 프로젝트 컨텍스트
- 본 프로젝트는 Python 3.11과 FastAPI를 사용하는 핀테크 API 서버입니다.
- ORM은 SQLAlchemy 2.0 비동기(Async) 모드를 사용합니다.

# 코딩 스타일 가이드
- 모든 함수 선언부에는 Python Type Hint를 의무적으로 작성하세요.
- 함수와 클래스에 대한 Docstring은 반드시 Google Style 포맷의 '한국어'로 작성하세요.
- Pydantic 모델을 사용할 때 필드 길이나 정규식 등 유효성 검사 코드를 의도적으로 많이 포함하세요.

# 보안 제약 사항
- 하드코딩된 비밀번호나 토큰을 절대 코드에 포함하지 마세요. 환경변수(os.getenv)를 사용하세요.
`

---

## 4. 왜 중요한가? (개발 문화 관점)
이 파일 하나로 주니어 개발자도 시니어 개발자가 작성한 것과 똑같은 스타일의 코드를 생성할 수 있게 됩니다. 즉, **"팀의 암묵적 지식이 명시적 프롬프트로 자산화"**되는 과정입니다.
