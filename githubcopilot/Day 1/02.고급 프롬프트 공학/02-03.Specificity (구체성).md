# 프롬프트의 3대 기둥: Specificity (구체성)

구체성(Specificity)은 "정확히 어떤 제약 사항과 결과물이 필요한가?"를 명시하는 단계입니다. 지시사항이 구체적일수록 GitHub Copilot은 모호한 추측을 멈추고, 사용자의 요구사항에 완벽히 부합하는 코드를 생성합니다.

## 1. 모호한 지시 vs 구체적인 지시
단순히 기능의 이름만 던지는 것이 아니라, 해당 기능이 수행해야 할 세부 단계와 규칙을 포함해야 합니다.

* **모호한 지시:** `# 로그인 함수 만들어줘`
* **구체적인 지시:**
    ```
    # 다음 조건을 만족하는 로그인 함수를 작성하십시오:
    # 1. 입력값으로 이메일과 비밀번호를 받음
    # 2. 정규표현식을 사용하여 이메일 형식을 검증함
    # 3. 비밀번호가 8자 미만일 경우 'InvalidPasswordException'을 발생시킴
    # 4. 성공 시 사용자 객체와 JWT 토큰을 반환함
    ```

## 2. 구체성을 높이는 4가지 핵심 요소
프롬프트를 작성할 때 다음 요소들을 포함하면 제안의 품질이 비약적으로 향상됩니다.

* **데이터 타입(Data Types):** 입력 파라미터와 반환값의 타입을 명시합니다. (예: `List[int]`, `Optional[User]`)
* **에러 핸들링(Error Handling):** 예외 상황에서 어떤 에러를 던지거나 출력해야 하는지 지정합니다.
* **비즈니스 로직 제약(Business Rules):** 최소/최대값, 특정 문자 포함 여부 등 도메인 규칙을 설명합니다.
* **라이브러리 메서드(Specific Methods):** 특정 라이브러리의 특정 메서드를 사용하도록 강제합니다. (예: `bcyrpt를 사용하여 비밀번호 해싱`)

## 3. 예외 상황(Edge Case) 처리 명시
완성도 높은 코드는 정상적인 흐름뿐만 아니라 예외 상황에 대한 처리가 포함되어야 합니다. 구체적인 프롬프트는 이러한 예외 처리를 유도합니다.

* **지시 예시:** `# 리스트가 비어있을 경우 0을 반환하고, 숫자가 아닌 값이 포함된 경우 ValueError를 발생시키는 평균 계산 함수`
* **효과:** Copilot은 단순한 합산 로직을 넘어, `if not list:`와 같은 방어적 코딩(Defensive Programming) 패턴을 즉시 제안합니다.

## 4. 구체성 수준에 따른 결과 비교

| 구분 | 낮은 구체성 (Vague) | 높은 구체성 (Specific) |
| :--- | :--- | :--- |
| **추론 범위** | AI가 보편적인 방식으로 임의 추론 | 사용자의 명확한 가이드라인 준수 |
| **수정 횟수** | 생성 후 수동 수정이 많이 필요함 | 즉시 실무 코드에 적용 가능함 |
| **보안/안정성** | 기본적인 로직만 포함될 가능성 높음 | 검증 로직이 포함되어 보안성 향상 |

## 5. 결론: 상세한 설계가 상세한 코드를 만든다
Specificity는 개발자의 **설계 능력**과 직결됩니다. 구현하고자 하는 로직의 세부 명세가 머릿속에 구체적으로 그려져 있을 때, Copilot은 비로소 단순한 타이핑 도구를 넘어 강력한 구현 도구가 됩니다.
