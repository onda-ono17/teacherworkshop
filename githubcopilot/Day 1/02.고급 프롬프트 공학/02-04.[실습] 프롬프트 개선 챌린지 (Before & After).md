# 실습: 프롬프트 개선 챌린지 (Before & After)

본 실습은 모호한 지시(Before)를 프롬프트의 3대 기둥을 활용하여 정교한 지시(After)로 개선함으로써, GitHub Copilot이 생성하는 코드의 질이 어떻게 변화하는지 체감하는 과정입니다.

---

## 챌린지 1: 구체성(Specificity) 개선하기

**상황:** 사용자의 입력을 받아 데이터베이스에 저장하기 전 검증하는 로직이 필요합니다.
```
### [Before] 모호한 프롬프트
# 이메일 유효성을 검사하는 함수 작성해줘

### [After] 개선된 프롬프트 (Specificity 적용)
# 다음 조건을 만족하는 이메일 유효성 검사 함수를 작성하십시오:
# 1. 정규표현식을 사용하여 이메일 형식을 확인합니다.
# 2. 이메일 호스트가 'example.com'인 경우 허용하지 않습니다.
# 3. 유효하지 않은 형식일 경우 'InvalidEmailError' 커스텀 예외를 발생시킵니다.
# 4. 유효할 경우 True를 반환합니다.
```
---

## 챌린지 2: 맥락(Context) 활용하기

**상황:** 프로젝트 내 정의된 특정 데이터 모델(User)을 사용하는 로직을 작성해야 합니다.
```
### [Before] 맥락이 없는 프롬프트
# 사용 정보를 데이터베이스에 저장하는 코드 작성

### [After] 개선된 프롬프트 (Context 활용)
1. **사전 준비:** `models.py` 탭을 열어 `User` 클래스 정의(id, email, username, created_at 필드)를 확인합니다.
2. **프롬프트 입력:**
# 열려 있는 models.py의 User 클래스 구조를 참조하여, 
# 새로운 사용자 객체를 생성하고 DB 세션에 추가(add)한 뒤 커밋(commit)하는 함수를 작성하십시오.
```
---

## 챌린지 3: 패러다임(Paradigm) 및 스타일 지정하기

**상황:** 웹 API 엔드포인트를 만들어야 하는데, 팀의 표준 프레임워크와 스타일을 준수해야 합니다.
```
### [Before] 패러다임이 없는 프롬프트
# 데이터 리스트를 반환하는 API 엔드포인트 만들어줘

### [After] 개선된 프롬프트 (Paradigm 적용)
# FastAPI 프레임워크를 사용하고 다음 스타일을 준수하십시오:
# 1. async/await를 사용하는 비동기 함수로 작성합니다.
# 2. 반환 타입은 List[UserResponse] Pydantic 모델을 사용합니다.
# 3. 의존성 주입(Depends)을 통해 데이터베이스 세션을 주입받습니다.
# 4. HTTP 200 상태 코드와 함께 데이터를 반환합니다.
```
