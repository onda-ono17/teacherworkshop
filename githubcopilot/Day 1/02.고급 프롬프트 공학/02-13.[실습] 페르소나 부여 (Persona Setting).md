# 실습: 페르소나 부여 (Persona Setting)

본 실습은 프롬프트 상단에 특정 역할(Persona)을 명시함에 따라 GitHub Copilot이 생성하는 코드의 보안 수준, 가독성, 그리고 설계 철학이 어떻게 변화하는지 확인하는 과정입니다.

---

## 1. 실습 환경 준비

실습을 위해 빈 파이썬 파일(`security_lab.py`) 또는 자바스크립트 파일(`security_lab.js`)을 준비합니다. 별도의 사전 코드는 필요하지 않으며, 프롬프트의 지시어만으로 결과의 차이를 비교합니다.

---

## 2. 실습 진행 시나리오

동일한 기능을 요구하되, '누가' 작성하는지에 대한 페르소나를 다르게 부여하여 제안되는 코드의 질을 비교합니다.

**시나리오 A: 일반적인 기능 요청 (기본 상태)**
1. 편집기에 아래 주석을 입력하고 제안을 확인합니다.
   ```python
   # 사용자 ID를 입력받아 데이터베이스에서 회원 정보를 조회하는 함수 작성
   ```
2. **확인 포인트:** - 코드가 단순히 문자열 포맷팅(f-string 등)을 사용하여 SQL 쿼리를 생성하는가?
   - 입력값에 대한 유효성 검사 로직이 포함되어 있는가?

**시나리오 B: 보안 전문가 페르소나 부여 (Persona)**
1. 기존 코드를 삭제하고, 아래와 같이 페르소나와 구체적인 성격을 정의한 주석을 입력합니다.
   ```python
   # 너는 보안에 매우 엄격한 시니어 백엔드 개발자야.
   # SQL 인젝션 공격을 완벽히 방어해야 하며, 클린 코드 원칙을 준수해야 해.
   # 사용자 ID를 입력받아 데이터베이스에서 회원 정보를 조회하는 함수를 작성해줘.
   ```
2. **확인 포인트:** - **SQL 파라미터화 (Prepared Statement):** 쿼리에 변수를 직접 넣지 않고 파라미터 바인딩 방식을 사용하는가?
   - **입력값 검증:** ID가 유효한 형식인지 확인하는 로직이 추가되었는가?
   - **예외 처리:** 데이터가 없을 때나 DB 오류 발생 시 보안상 안전한 에러 메시지를 반환하는가?
