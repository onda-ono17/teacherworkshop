# 실습: 사내 컨벤션 준수를 위한 Few-shot 프롬프팅

본 실습은 일반적인 코드가 아닌, 특정 기업의 개발 표준(Convention)이나 고유한 데이터 형식을 GitHub Copilot에게 학습시켜 일관성 있는 코드를 생성하도록 유도하는 Few-shot 기법을 익히는 데 목적이 있습니다.

---

## 1. 실습 개요
* 목표: 1~2개의 예시 코드를 제공하여 Copilot이 사내 표준 패턴을 파악하고, 동일한 규칙으로 다음 코드를 작성하게 만듭니다.
* 준비물: Visual Studio Code 또는 JetBrains IDE, GitHub Copilot 확장 프로그램.
* 언어: Python 또는 JavaScript.

---

## 2. 실습 과제 목록

### 과제 1: 사내 표준 로그 포맷 적용 (Log Formatting)
상황: ABC 테크의 보안 정책에 따라 모든 로그는 [시스템명][날짜][레벨] 메시지 | 처리시간 형식을 유지해야 합니다.

* 수행 방법: 아래 예시 코드를 편집기에 입력하고 마지막 줄에서 Copilot의 제안을 기다립니다.

```python
# 사내 로그 표준에 따라 메시지를 생성하는 함수 예시:
# 예시 1: 입력: "사용자 인증", 레벨: "INFO", 시간: "5ms" -> 결과: "[MY-APP][2026-03-14][INFO] 사용자 인증 | 5ms"
# 예시 2: 입력: "DB 연결 실패", 레벨: "ERROR", 시간: "120ms" -> 결과: "[MY-APP][2026-03-14][ERROR] DB 연결 실패 | 120ms"
# 과제: 입력: "데이터 캐싱 성공", 레벨: "DEBUG", 시간: "2ms" -> 결과:
```

* 확인 포인트: Copilot이 앞선 예시의 대괄호 형식, 날짜 및 구분자(|)를 그대로 복제하여 생성하는지 확인합니다.

---

### 과제 2: 레거시 데이터 변환 (Data Transformation)
상황: 사내 구형 시스템에서 내려주는 콤마 구분 데이터를 신규 시스템용 JSON 객체로 변환해야 합니다. 이때 키(Key) 이름은 반드시 사내 공통 용어 사전(Vocabulary)을 따라야 합니다.

* 수행 방법: 아래 패턴을 입력하십시오.

```javascript
// 레거시 데이터를 사내 표준 객체로 변환하는 예시
// 원본: "USER_01, 홍길동, 29, 서울" -> 변환: { uid: "USER_01", name: "홍길동", age_val: 29, loc_info: "서울" }
// 원본: "USER_02, 김철수, 34, 부산" -> 변환: { uid: "USER_02", name: "김철수", age_val: 34, loc_info: "부산" }
// 원본: "USER_03, 이영희, 25, 광주" -> 변환:
```

* 확인 포인트: age 대신 age_val, location 대신 loc_info라는 특수한 키 이름을 Copilot이 정확히 찾아내는지 확인합니다.

---

### 과제 3: 보안 헤더 유효성 검사 (Security Validation)
상황: 모든 내부 API 요청은 특정 규칙을 가진 헤더(X-INTERNAL-AUTH)를 포함해야 하며, 이에 대한 유효성 검사 로직을 동일한 스타일로 작성해야 합니다.

* 수행 방법: 아래 내용을 입력하십시오.

```python
# 보안 헤더 검증 패턴 예시
# 패턴 1: header가 'INT-'로 시작하면 return True
# 패턴 2: header가 'EXT-'로 시작하면 return False
# 패턴 3: header가 비어있으면 raise SecurityError("Empty Header")
# 과제: header가 'DEV-'로 시작하면
```

* 확인 포인트: Copilot이 앞서 사용된 return 방식이나 raise SecurityError와 같은 예외 처리 스타일을 그대로 유지하며 다음 조건을 완성하는지 확인합니다.
