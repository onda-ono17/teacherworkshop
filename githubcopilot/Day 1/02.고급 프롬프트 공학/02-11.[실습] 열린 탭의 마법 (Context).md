# 실습: "열린 탭"의 마법 (Context)

본 실습은 GitHub Copilot이 현재 작업 중인 파일뿐만 아니라 IDE 상단에 열려 있는 주변 탭(Neighboring Tabs)의 정보를 어떻게 참조하여 코드의 정확도를 높이는지 직접 확인하는 과정입니다.

---

## 1. 실습 환경 준비

먼저 프로젝트의 기준이 되는 데이터 모델을 정의합니다. 아래 코드를 사용하여 파일을 생성하십시오.

**파일명: user_model.py**
```python
from datetime import datetime
from dataclasses import dataclass

@dataclass
class User:
    user_uuid: str
    official_email: str
    display_name: str
    is_active: bool
    joined_at: datetime
```

---

## 2. 실습 진행 단계

**단계 1: 맥락 차단 상태에서의 생성 (Before)**
1. 위에서 작성한 user_model.py 탭을 IDE 상단에서 완전히 닫습니다.
2. 새로운 파일 main.py를 생성합니다.
3. main.py의 첫 줄에 아래 주석을 입력하고 Enter를 누릅니다.
   ```python
   # user_model.py의 User 클래스를 사용하여 새로운 사용자를 데이터베이스에 저장하는 함수 작성
   ```
4. 결과 확인: Copilot이 제안하는 코드의 필드명을 확인합니다. 일반적으로 id, name, email 등 보편적인 필드명을 추측하여 생성할 것입니다.

**단계 2: 맥락 공유 상태에서의 생성 (After)**
1. 닫았던 user_model.py 파일을 다시 열어 화면에 활성화(탭이 표시된 상태)합니다.
2. main.py로 돌아와 이전 단계에서 생성된 코드를 모두 삭제합니다.
3. 동일한 주석을 다시 입력하고 제안을 기다립니다.
   ```python
   # user_model.py의 User 클래스를 사용하여 새로운 사용자를 데이터베이스에 저장하는 함수 작성
   ```
4. 결과 확인: 이번에는 Copilot이 user_uuid, official_email, display_name 등 user_model.py에 실제 정의된 고유 필드명과 타입을 정확히 인용하여 코드를 제안하는지 확인합니다.
