# 실습: Zero vs Few-shot 배틀 (데이터 파싱)

본 실습은 아무런 예시 없이 지시만 내리는 Zero-shot 방식과 구체적인 입출력 예시를 제공하는 Few-shot 방식의 결과물 차이를 비교하여, 정교한 프롬프팅이 실제 업무 효율에 미치는 영향을 학습하는 과정입니다.

---

## 1. 실습 환경 준비

실습을 위해 아래의 가상 로그 데이터가 포함된 파일을 생성하거나, 코드를 작성할 준비를 합니다. 이 로그 데이터는 표준적이지 않은 형식을 포함하고 있어 AI의 추론 능력을 시험하기에 적합합니다.

**대상 데이터 예시 (server.log):**
2026-03-14 10:20:05 [ERROR] Connection timeout at segment 4
2026-03-14 10:21:30 [INFO] Heartbeat signal sent
2026-03-14 10:22:15 [ERROR] Invalid user credentials for 'admin'

---

## 2. 실습 진행 시나리오

수강생을 A그룹과 B그룹으로 나누어 진행하거나, 개별적으로 아래 두 단계를 순차 수행하여 결과를 비교합니다.

**시나리오 A: Zero-shot 방식 (단순 지시)**
1. `main.py` 또는 `main.js` 파일에 아래 주석만 입력하고 제안을 확인합니다.
   ```python
   # 로그 파일에서 날짜와 에러 메시지만 추출하여 JSON 리스트로 변환하는 함수 작성
   ```
2. **확인 포인트:** - Copilot이 생성한 정규표현식이 로그의 날짜와 메시지 영역을 정확히 구분하는가?
   - [ERROR] 레벨만 정확히 필터링하는 로직이 포함되었는가?
   - 결과물의 JSON 키 이름이 사용자의 의도와 일치하는가?

**시나리오 B: Few-shot 방식 (예시 포함 지시)**
1. 기존 코드를 삭제하고, 아래와 같이 구체적인 입출력 예시를 포함한 주석을 입력합니다.
   ```python
   # 로그 파싱 규칙 예시:
   # 입력: 2026-03-14 10:20:05 [ERROR] Connection timeout at segment 4
   # 결과: { "log_date": "2026-03-14", "msg_content": "Connection timeout at segment 4" }
   # 위 예시 형식을 참고하여, 로그 파일 내 모든 [ERROR] 항목을 JSON 리스트로 변환하는 함수를 작성하십시오.
   ```
2. **확인 포인트:** - Copilot이 예시에서 정의한 특정 키 이름("log_date", "msg_content")을 그대로 사용하는가?
   - 예시에서 시간을 제외했으므로, 생성된 코드도 날짜만 추출하도록 정규표현식이 조정되었는가?
