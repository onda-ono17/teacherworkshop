# GitHub Copilot 프롬프트 엔지니어링 5계명

GitHub Copilot은 마법 지팡이가 아니라 정교한 비행 보조 장치입니다. 개발자가 유능한 기장(Pilot)으로서 Copilot을 올바르게 제어하기 위해 반드시 지켜야 할 5가지 원칙을 정리합니다.

---

## 제1계명: 맥락의 힘을 믿으라 (Context is King)
Copilot은 당신의 IDE 환경을 보고 있습니다. 관련 있는 데이터 모델, 유틸리티 함수, 인터페이스 정의서가 담긴 파일들을 탭에 열어두십시오. 관련 없는 탭은 닫아 Copilot이 엉뚱한 맥락을 참고하지 않도록 관리하는 것도 기장의 역할입니다.

## 제2계명: 구체적일수록 결과는 정교해진다 (Be Specific)
모호한 요청은 모호한 코드를 만듭니다. 입력값의 타입, 반환 형식, 예외 처리 규칙, 사용할 특정 라이브러리를 명시하십시오. "무엇을"뿐만 아니라 "어떻게"와 "제약 사항"이 포함될 때 비로소 실무 수준의 코드가 생성됩니다.

## 제3계명: 예시로 가르치라 (Show, Don't Just Tell)
백 마디 설명보다 한 줄의 예시 코드가 더 강력합니다. 원하는 스타일의 코드를 한 줄 직접 작성하거나, 주석으로 입출력 예시를 제공하십시오. Copilot은 당신의 코딩 스타일과 패턴을 즉시 학습하여 그대로 복제할 것입니다.

## 제4계명: 한 번에 포기하지 말고 정제하라 (Iterate and Refine)
첫 번째 제안이 완벽하지 않다면 프롬프트를 수정하십시오. 요구사항을 더 작게 쪼개거나, 다른 단어를 사용하여 다시 질문하십시오. Copilot과의 대화는 일회성 명령이 아니라 최적의 해답을 찾아가는 협업 과정입니다.

## 제5계명: 최종 검증은 당신의 몫이다 (Trust but Verify)
Copilot이 생성한 코드가 논리적으로 타당한지, 보안 취약점은 없는지, 프로젝트의 전체 흐름과 맞는지는 항상 개발자가 판단해야 합니다. 생성된 코드에 대한 책임은 AI가 아닌 도구를 사용한 개발자에게 있음을 잊지 마십시오.

---

## 요약: 성공적인 프롬프팅을 위한 공식
**최적의 결과 = 명확한 목표 + 풍부한 맥락 + 구체적인 제약 조건**
