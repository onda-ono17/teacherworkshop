### 블랙박스 테스트 기법: 경계값 분석 (Boundary Value Analysis, BVA)

경계값 분석은 프로그램의 결함이 주로 입력 영역의 '경계'에서 발생한다는 통계적 특성에 착안하여, 경계값과 그 주변 값을 집중적으로 테스트하는 기법입니다. 동등 분할 기법을 보완하는 가장 강력한 테스트 설계 방법 중 하나입니다.

---

#### 1. 주요 개념 및 목적
* **개념:** 입력 조건의 경계 부분(최소값, 최대값) 및 그 직전/직후 값을 테스트 데이터로 선택합니다.
* **목적:** 개발자가 흔히 저지르는 비교 연산자 오류(> 와 >=의 혼용), 루프의 반복 횟수 오류(Off-by-one error) 등을 효과적으로 찾아냅니다.

---

#### 2. 테스트 값 선정 기준 (5-Point Testing)
하나의 유효 영역이 [Min, Max]일 때, 일반적으로 다음과 같은 5가지 지점을 테스트합니다.

* **최소값 (Min):** 하한 경계값 그 자체.
* **최소값 직후 (Min+):** 하한 경계 바로 안쪽의 값.
* **중간값 (Nominal):** 영역 내의 지극히 정상적인 값.
* **최대값 직전 (Max-):** 상한 경계 바로 안쪽의 값.
* **최대값 (Max):** 상한 경계값 그 자체.



---

#### 3. 실습 사례: 할인 대상 연령 체크 함수

**입력 조건:** "만 19세 이상부터 만 65세 이하까지 할인 혜택 적용"

| 테스트 케이스 ID | 입력값 (나이) | 구분 | 예상 결과 |
| :--- | :--- | :--- | :--- |
| TC_BVA_01 | 18 | 유효하지 않은 경계 (Min-) | 할인 대상 아님 |
| **TC_BVA_02** | **19** | **최소 경계값 (Min)** | **할인 대상** |
| TC_BVA_03 | 20 | 유효 영역 내부 (Min+) | 할인 대상 |
| TC_BVA_04 | 40 | 중간값 (Nominal) | 할인 대상 |
| TC_BVA_05 | 64 | 유효 영역 내부 (Max-) | 할인 대상 |
| **TC_BVA_06** | **65** | **최대 경계값 (Max)** | **할인 대상** |
| TC_BVA_07 | 66 | 유효하지 않은 경계 (Max+) | 할인 대상 아님 |

---

#### 4. GitHub Copilot 활용 실습: /tests 명령어와 BVA

수강생은 Copilot이 경계값 분석을 얼마나 정확하게 수행하는지 실습을 통해 확인합니다.

**단계 1: 의도적으로 오류가 포함된 코드 작성**
```python
def is_eligible_for_discount(age):
    # 원래 19세 이상이어야 하지만, 실수로 > 19로 작성함 (오류 발생 지점)
    if age > 19 and age <= 65:
        return True
    return False
```

**단계 2: Copilot에게 BVA 기반 테스트 요청**
측면 채팅창(Copilot Chat)에 다음과 같이 구체적으로 요청합니다.
> "is_eligible_for_discount 함수에 대해 경계값 분석(BVA) 기법을 사용하여 테스트 코드를 작성해줘. 19세와 65세 지점의 경계값을 반드시 포함해줘."

**단계 3: 결과 확인 및 버그 수정**
* Copilot이 생성한 테스트 코드 중 test_age_19가 실패(Fail)하는 것을 확인합니다.
* 실패 원인이 코드의 > 연산자 때문임을 파악하고 >=로 수정하는 과정을 거칩니다.
