# Hands on Lab: 예외 상황 및 TDD 시나리오 설계

AI를 통해 누락되기 쉬운 기획적/기술적 결함을 미리 발견하고, 이를 기반으로 테스트 시나리오를 설계하는 실습입니다.

---

## 1. 개요 (Context)
*   **문제 상황:** 간단한 "할인 금액 계산 함수"를 작성했지만, 사용자 입력의 다양성에 따른 예외 처리가 전혀 고려되지 않았습니다.
*   **목표:** Copilot에게 엣지 케이스(Edge Cases) 추출을 요청하고, 그 조건들을 검증하는 테스트 시나리오를 생성합니다.

---

## 2. 실습 진행 (Action)

1.  빈 파일 `discount_calculator.py`를 만들고 아래 코드를 붙여넣습니다.

```python
def calculate_discount(price, discount_rate):
    """
    상품 가격과 할인율(%)이 주어졌을 때, 최종 지불 금액을 반환합니다.
    """
    discount_amount = price * (discount_rate / 100)
    final_price = price - discount_amount
    return final_price
```

2.  **엣지 케이스 분석 요청:** Copilot Chat 창에 다음과 같이 입력합니다.
    > "이 할인 금액 계산 로직에서 논리적 결함이나 개발자가 고려해야 할 엣지 케이스(예: 음수 처리, 100% 이상의 할인율 등)를 5가지 정도 찾아줘."

    *예상 답변 관찰:*
    - `price`나 `discount_rate`가 음수인 경우
    - `discount_rate`가 100을 초과하는 경우
    - 파라미터로 문자열이나 None이 들어오는 자료형 에러
    - 부동소수점 오차 문제 (예: 99.99원)

3.  **단위 테스트 뼈대 자동 생성:** 엣지 케이스 분석 결과를 바탕으로, 이를 검증하기 위한 테스트 코드를 바로 만들어달라고 요청합니다.
    > "위에서 도출된 5가지 엣지 케이스를 검증하는 pytest 형태의 테스트 함수들을 만들어줘."

4.  생성된 `test_discount_calculator.py` 코드를 워크스페이스에 새로 만들어 복사-붙여넣기 합니다.

---

## 3. 실무 팁 (Best Practices)
*   **TDD(Test-Driven Development) 적용:** 위 실습처럼 테스트 코드(에러 케이스들)를 먼저 확보한 뒤, 이 테스트들이 전부 통과할 때까지(Pass) 실제 로직(calculate_discount)에 if문과 Exception 핸들링 코드를 추가해나가는 방식이 Copilot 환경에서 가장 생산성 높은 개발 워크플로우 중 하나입니다.
