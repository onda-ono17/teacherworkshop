### 실습: 복잡한 할인율 계산기 로직 및 테스트 설계

본 실습은 복잡한 중복 할인 조건과 시스템 제한 사항이 포함된 비즈니스 로직을 분석하고, GitHub Copilot을 활용하여 동등 분할 및 경계값 분석 기반의 완성도 높은 테스트 수트를 구축하는 것을 목표로 합니다.

---

#### 1. 비즈니스 로직 및 규칙 (Business Rules)

수강생은 아래의 규칙을 바탕으로 할인율 계산 함수를 구현하고 검증해야 합니다.

**구매 금액별 기본 할인:**
* 50,000원 미만: 0% 할인
* 50,000원 이상 ~ 100,000원 미만: 5% 할인
* 100,000원 이상: 10% 할인

**대상별 추가 할인 (기본 할인과 중복 적용 가능):**
* 19세 미만 (청소년): 전체 금액의 10% 추가 할인
* 65세 이상 (시니어): 전체 금액의 15% 추가 할인
* 해당 사항 없음 (일반): 추가 할인 없음

**시스템 제한 및 제약 조건:**
* 한 번에 주문 가능한 최대 수량은 100개로 제한합니다. (100개 초과 시 에러)
* 구매 금액 및 주문 수량은 음수가 될 수 없습니다.

---

#### 2. 에러 시나리오 체크리스트 (수강생 배포용 템플릿)

요구사항을 분석하여 작성된 테스트 명세서입니다. 수강생은 이 리스트를 바탕으로 Copilot에게 테스트 코딩을 지시합니다.

| 테스트 유형 | 입력 항목 | 테스트 값 (데이터) | 기대 결과 (Expected Result) |
| :--- | :--- | :--- | :--- |
| 경계값 (금액) | 구매 금액 | 49,999원 / 50,000원 | 0% 적용 / 5% 적용 확인 |
| 경계값 (금액) | 구매 금액 | 99,999원 / 100,000원 | 5% 적용 / 10% 적용 확인 |
| 경계값 (연령) | 사용자 나이 | 18세 / 19세 | 청소년 할인 적용 / 미적용 확인 |
| 경계값 (연령) | 사용자 나이 | 64세 / 65세 | 일반 요금 / 시니어 할인 적용 확인 |
| 경계값 (수량) | 주문 수량 | 100개 / 101개 | 주문 성공 / '최대 수량 초과' 에러 |
| 에러 처리 | 구매 금액 | -1원 (음수) | '유효하지 않은 금액' 에러 발생 |
| 에러 처리 | 사용자 나이 | 0세 / 150세 | 정상 처리 혹은 시스템 정책에 따른 에러 |
| 데이터 타입 | 구매 금액 | "abc" (문자열) | 입력 형식 오류 (Type Error) 처리 |
| 데이터 타입 | 구매 금액 | 빈 값 (Null) | '필수 값 누락' 에러 메시지 |

---

#### 3. GitHub Copilot 활용 실습 가이드

수강생들은 각 단계별로 Copilot의 대화형 기능을 활용하여 생산성을 높이는 연습을 수행합니다.

**활동 1: 테스트 케이스 도출 자동화**
* **방법:** 비즈니스 로직 주석을 파일 상단에 작성한 후, Copilot Chat을 활성화합니다.
* **프롬프트:** "위 비즈니스 로직에 대해 경계값 분석과 동등 분할 기법을 적용해서 테스트 케이스를 표 형식으로 정리해줘. 누락된 예외 상황이 있다면 추가해줘."

**활동 2: 테스트 데이터셋 생성 (JSON/List)**
* **방법:** 별도의 데이터 파일이나 테스트 코드 내에서 샘플 데이터를 요청합니다.
* **프롬프트:** "할인율 계산기를 테스트하기 위한 유효한 데이터셋 5개와 경계값 에러 데이터셋 3개를 JSON 형식으로 만들어줘. 각 데이터에는 테스트 의도를 주석으로 달아줘."

**활동 3: /tests 명령어를 이용한 검증 코드 작성**
* **방법:** 구현된 함수 블록을 드래그하여 선택한 뒤 명령어를 입력합니다.
* **프롬프트:** "/tests 이 함수의 경계값(49,999원, 50,000원 등)과 중복 할인 조건(청소년+10만원 이상)을 포함한 유닛 테스트를 Pytest로 작성해줘."
