### 실습: 동등 분할 및 경계값 분석 기반 테스트 설계

본 실습은 동등 분할(EP)과 경계값 분석(BVA) 기법을 하나의 시나리오에 통합하여 적용함으로써, 논리적인 테스트 케이스 설계 능력과 GitHub Copilot 활용 역량을 동시에 강화하는 것을 목적으로 합니다.

---

#### 1. 실습 시나리오: 스마트 배송료 계산 시스템

배송 시스템의 무게별 운임 계산 함수를 구현하고 검증해야 합니다. 다음은 비즈니스 요구사항입니다.

* **입력 조건:** 화물의 무게(kg, 소수점 첫째 자리까지 허용)
* **운임 규칙:**
    - 0.1kg 미만: 배송 불가 (에러 메시지 반환)
    - 0.1kg 이상 ~ 20.0kg 이하: 일반 배송 (10,000원)
    - 20.1kg 이상 ~ 50.0kg 이하: 대형 배송 (25,000원)
    - 50.0kg 초과: 특수 배송 (별도 문의 안내 메시지 반환)
* **제약:** 무게는 양수여야 하며, 500.0kg을 초과하는 입력은 시스템 오류로 간주합니다.

---

#### 2. 미션 1: 동등 분할(EP) 영역 정의

수강생은 요구사항을 바탕으로 유효 영역과 비유효 영역을 구분하여 테이블을 작성합니다.

| 영역 ID | 구분 | 무게 범위(w) | 예상 결과 |
| :--- | :--- | :--- | :--- |
| EP_01 | 비유효 (하한) | w < 0.1 | 배송 불가 에러 |
| EP_02 | 유효 (일반) | 0.1 <= w <= 20.0 | 10,000원 |
| EP_03 | 유효 (대형) | 20.1 <= w <= 50.0 | 25,000원 |
| EP_04 | 유효 (특수) | 50.0 < w <= 500.0 | 별도 문의 안내 |
| EP_05 | 비유효 (상한) | w > 500.0 | 시스템 오류 |

---

#### 3. 미션 2: 경계값 분석(BVA) 테스트 케이스 설계

가장 결함이 많이 발생하는 경계 지점을 추출하여 구체적인 테스트 값을 선정합니다. (핵심 경계: 0.1, 20.0, 20.1, 50.0, 500.0)

| 테스트 ID | 입력 값 (w) | 선택 이유 | 예상 결과 |
| :--- | :--- | :--- | :--- |
| BVA_01 | 0.0 | 최소 경계 미만 (Invalid) | 배송 불가 |
| BVA_02 | 0.1 | 최소 경계값 (Valid Min) | 10,000원 |
| BVA_03 | 20.0 | 일반 배송 상한 (Valid Max) | 10,000원 |
| BVA_04 | 20.1 | 대형 배송 하한 (Valid Min) | 25,000원 |
| BVA_05 | 50.0 | 대형 배송 상한 (Valid Max) | 25,000원 |
| BVA_06 | 50.1 | 특수 배송 하한 (Valid Min) | 별도 문의 |
| BVA_07 | 500.0 | 시스템 허용 최대값 | 별도 문의 |
| BVA_08 | 500.1 | 시스템 허용 초과 | 시스템 오류 |

---

#### 4. 미션 3: GitHub Copilot을 활용한 구현 및 검증

수강생은 설계한 명세서를 바탕으로 Copilot과 협업하여 코드를 완성합니다.

**단계 1: 기초 코드 작성**
수강생이 직접 또는 Copilot의 도움을 받아 `shipping.py` 파일을 생성합니다.

**단계 2: 분석 요청 (측면 채팅)**
> "작성된 shipping.py 코드에 대해 동등 분할과 경계값 분석 기법을 적용하여 논리적 결함이 없는지 리뷰해줘. 특히 20.0kg과 20.1kg 사이의 경계 처리가 정확한지 확인해줘."

**단계 3: 자동 테스트 생성 (/tests)**
> "/tests 설계한 EP_01부터 EP_05, BVA_01부터 BVA_08까지의 케이스를 모두 포함하는 pytest 코드를 생성해줘. 각 테스트 함수명에 어떤 기법을 썼는지 주석으로 달아줘."
