# GitHub Copilot 핵심 명령어 및 프롬프트 치트 시트 (Cheat Sheet)

이 문서는 GitHub Copilot의 주요 기능을 빠르게 찾아보고 효율적으로 프롬프트를 작성하기 위한 요약 가이드입니다.

---

## 1. 필수 단축키 (Core Shortcuts)

| 기능 | Windows / Linux | macOS |
| :--- | :--- | :--- |
| **인라인 채팅 실행** | Ctrl + I | Cmd + I |
| **측면 채팅 호출** | Ctrl + Alt + I | Cmd + Shift + I |
| **제안 수락 (Accept)** | Tab | Tab |
| **다음 제안 보기** | Alt + ] | Option + ] |
| **이전 제안 보기** | Alt + [ | Option + [ |
| **전체 제안창 열기** | Ctrl + Enter | Ctrl + Enter |

---

## 2. 슬래시 명령어 (Slash Commands)

채팅창에 /를 입력하여 AI에게 명확한 태스크를 부여하십시오.

* **/explain:** 선택한 코드의 로직과 작동 원리를 단계별로 설명합니다.
* **/fix:** 선택한 코드의 오류를 분석하고 수정안을 제안합니다.
* **/tests:** 코드의 안정성 검증을 위한 단위 테스트(Unit Test)를 생성합니다.
* **/doc:** 함수나 클래스에 대한 표준 문서화 주석(Docstring)을 생성합니다.
* **/terminal:** 필요한 터미널 명령어를 확인하고 즉시 삽입합니다.
* **/new:** (@workspace와 함께 사용) 프로젝트의 기초 뼈대를 설계합니다.

---

## 3. 컨텍스트 제어 (Context Variables)

질문의 범위를 지정하여 답변의 정확도를 높이는 기법입니다.

* **@workspace:** 프로젝트 전체 구조를 바탕으로 질문합니다. (예: @workspace 인증 로직 위치 알려줘)
* **#file:** 특정 파일을 참조 대상으로 명시합니다. (예: #file:main.py 로직 분석해줘)
* **#selection:** 현재 드래그로 선택된 영역만 집중적으로 참고하도록 제한합니다.

---

## 4. 고품질 프롬프트 작성 3원칙

좋은 결과를 얻기 위해서는 프롬프트에 다음 세 가지 요소를 포함하십시오.

1. **역할(Role) 부여:** "너는 시니어 보안 전문가야", "너는 알고리즘 최적화 전문가야"
2. **구체적인 맥락(Context):** "FastAPI와 SQLAlchemy를 사용하는 환경에서...", "O(n) 복잡도로..."
3. **제약 사항(Constraint):** "외부 라이브러리 없이 순수 파이썬만 사용해", "Google 스타일 주석을 달아줘"

---

## 5. 상황별 권장 인터페이스

* **인라인 채팅 (Inline Chat):** 변수명 변경, 짧은 로직 수정, 주석 삽입 등 흐름을 깨지 않는 즉각적인 작업.
* **측면 채팅 (Sidebar Chat):** 코드 전체 리뷰, 아키텍처 설계, 복잡한 에러 분석 등 깊이 있는 대화가 필요한 작업.

---

## 6. 주의 사항 (Best Practices)

* **비판적 검토:** Copilot은 부조종사일 뿐입니다. 생성된 코드의 최종 검증과 책임은 개발자 본인에게 있습니다.
* **탭 관리:** 관련 있는 파일들을 IDE 상단 탭에 열어두는 것만으로도 Copilot의 제안 품질이 향상됩니다.
* **단위 테스트 병행:** 생성된 코드는 반드시 /tests 명령어를 통해 검증하는 습관을 기르십시오.
