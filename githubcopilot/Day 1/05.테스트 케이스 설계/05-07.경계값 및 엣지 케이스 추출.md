# 경계값 및 엣지 케이스(Edge Case) 추출

테스트 케이스를 작성할 때 가장 어려운 부분은 개발자가 "미처 생각하지 못한 예외 상황(Edge Case)"을 식별하는 것입니다. Copilot을 TDD의 파트너로 삼아 발생 가능한 엣지 케이스를 먼저 도출하는 기법을 알아봅니다.

---

## 1. AI와 함께하는 테스트 시나리오 브레인스토밍
개발자는 성공 스토리에 빠지기 쉽습니다. 따라서 로직을 구현하기 전에 AI에게 "취약점과 예외 케이스를 찾아내라"고 지시하는 것이 효과적입니다.

* **프롬프트 템플릿 예시:**
  > "다음 비밀번호 유효성 검사 로직(또는 기획 스펙)에서 내가 놓칠 수 있는 엣지 케이스나 보안 취약점을 5가지 이상 찾아줘. 경계값 분석(Boundary Value Analysis) 기법을 사용해."

## 2. 엣지 케이스를 도출하는 주요 패턴

1.  **자료형 및 입력 포맷 에러:**
    *   정수를 기대하는 곳에 부동 소수점, 문자열, Array 입력
    *   `None`, `Null`, `Undefined`, 빈 문자열(`""`) 입력
2.  **경계값 (Boundary Conditions):**
    *   배열의 길이가 0이거나 너무 방대한 메모리가 필요한 경우
    *   날짜 계산시 윤년(Leap year), 월말 계산 오류
3.  **동시성 및 타이밍 문제:**
    *   비동기 호출 중 타임아웃, 중복 클릭 시의 Race Condition
4.  **권한 및 보안 정책 우회:**
    *   Admin만 접근 가능한 필드에 일반 유저의 강제 주입(Injection) 시도

---

## 3. 테스트 코드 반영

추출된 엣지 케이스들을 단순히 읽고 끝내는 것이 아니라 일괄적으로 테스트 코드로 변환할 수 있습니다.
*   *적용 프롬프트:* "훌륭해. 네가 찾은 5개의 엣지 케이스를 모두 포함하는 `pytest` (혹은 `JUnit`) 형태의 테스트 스크립트를 작성해 줘."
