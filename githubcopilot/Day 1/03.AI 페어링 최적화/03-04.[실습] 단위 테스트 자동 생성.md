# 실습: 단위 테스트 자동 생성 (/tests)

본 실습은 GitHub Copilot의 슬래시 명령어인 /tests를 활용하여 기존 코드에 대한 단위 테스트(Unit Test)를 자동으로 설계하고, 다양한 입력 값에 대한 코드의 안정성을 검증하는 과정을 학습합니다.

---

## 1. 준비: 테스트 대상 코드 작성

테스트의 대상이 될 함수를 작성합니다. 아래 함수는 사용자 비밀번호의 유효성을 세 가지 규칙(길이, 숫자 포함, 대문자 포함)에 따라 검사합니다.

**파일명: auth_utils.py**
```python
def validate_password(password):
    # 1. 길이 체크 (8자 이상)
    if len(password) < 8:
        return False
    
    # 2. 숫자 포함 여부 체크
    if not any(char.isdigit() for char in password):
        return False
    
    # 3. 대문자 포함 여부 체크
    if not any(char.isupper() for char in password):
        return False
    
    return True
```

---

## 2. 과정: /tests 명령어를 통한 테스트 코드 생성

수강생은 Copilot Chat을 활용하여 수동으로 작성하기 번거로운 테스트 케이스를 순식간에 생성합니다.

**단계 1: 함수 선택 및 명령어 입력**
1. auth_utils.py 파일에서 validate_password 함수 블록 전체를 마우스로 드래그하여 선택합니다.
2. 측면 채팅창(Copilot Chat)을 열고 /tests 라고 입력한 뒤 전송합니다.

**단계 2: 제안된 테스트 코드 검토**
* 확인 포인트: Copilot이 아래와 같은 다양한 시나리오를 자동으로 생성했는지 확인합니다.
    - 정상적인 비밀번호 (예: StrongPass123)
    - 8자 미만의 짧은 비밀번호
    - 숫자가 누락된 비밀번호
    - 대문자가 누락된 비밀번호
    - 빈 문자열이나 특수문자만 있는 경우

**단계 3: 테스트 파일 저장 및 실행**
1. Copilot이 제안한 코드를 복사하여 새로운 파일 test_auth_utils.py로 저장합니다.
2. 터미널에서 pytest 또는 unittest를 사용하여 테스트를 실행하고 결과를 확인합니다.
   ```bash
   # pytest가 설치된 경우
   pytest test_auth_utils.py
   ```

## 3. 결과: 생성된 테스트 코드 예시

Copilot은 일반적으로 아래와 같이 pytest 프레임워크를 활용한 효율적인 테스트 코드를 제안합니다.

```python
import pytest
from auth_utils import validate_password

def test_validate_password_success():
    assert validate_password("Python2026!") is True

def test_validate_password_too_short():
    assert validate_password("Short1!") is False

def test_validate_password_no_digit():
    assert validate_password("NoDigitPassword") is False

def test_validate_password_no_uppercase():
    assert validate_password("lowercase123") is False

def test_validate_password_empty():
    assert validate_password("") is False
```
