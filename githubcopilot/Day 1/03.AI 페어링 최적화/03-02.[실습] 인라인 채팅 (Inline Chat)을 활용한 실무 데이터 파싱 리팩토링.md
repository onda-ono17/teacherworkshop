# 실습: 인라인 채팅(Inline Chat)을 활용한 실무 "데이터 파싱" 리팩토링

본 실습은 의미 없는 알고리즘 리팩토링을 벗어나, 실무에서 자주 마주치는 **"더러운 외부 데이터(Raw Text)를 깔끔한 오브젝트로 파싱(Parsing)"**하는 귀찮은 과정을 인라인 채팅으로 순식간에 해결하는 경험을 제공합니다.

## 1. 실습 환경 준비

""

def parse_user_data(text):
    # TODO: 아래 형편없는 수동 파싱 코드를 인라인 채팅으로 리팩토링하세요.
    result = []

## 2. 실습 과제 수행 단계

수강생은 편집기에서 인라인 채팅(Ctrl + I 또는 Cmd + I)을 호출하여 아래 과제들을 순차적으로 수행합니다.

### 과제 1: 정규식(Regex) 기반의 스마트 파싱으로 변경
1. parse_user_data 함수 내부 전체를 마우스로 드래그하여 선택합니다.
2. 인라인 채팅창에 다음과 같이 입력합니다.
   > "현재 코드는 구분자(|, /, ,)에 따라 일일이 분기처리하고 있어 비효율적이야. re 모듈(정규식)을 사용해서 어떤 구분자가 오든 한 번에 파싱되는 간결한 코드로 리팩토링해줘."
3. **확인 포인트:** 지저분했던 if-elif 블록이 사라지고 
e.split() 등을 활용한 깔끔한 파싱 코드로 변환되었는지 확인하고 Accept 합니다.

### 과제 2: 타입 힌트와 Pydantic (또는 Dataclass) 제안받기
1. 바뀐 함수 전체를 다시 선택합니다.
2. 인라인 채팅창에 입력합니다.
   > "현재 딕셔너리({})를 반환하고 있는데, 오타 방지를 위해 파이썬 내장 dataclass 구조체를 하나 정의하고 그걸 반환하도록 변경해줘. 타입 힌트도 포함해줘."
3. **확인 포인트:** 코드 상단에 @dataclass class User: 구조체가 자동으로 생성되고, 함수의 반환 형태가 딕셔너리에서 User 객체의 리스트로 고급화되었는지 확인합니다.

## 3. 최종 결과 예시 (After)

모든 리팩토링을 마친 후의 코드는 다음과 같은 형태를 띠게 됩니다.

`python
import re
from dataclasses import dataclass
from typing import List

@dataclass
class User:
    name: str
    age: int
    phone: str
    dept: str

def parse_user_data(text: str) -> List[User]:
    """일관성 없는 텍스트 데이터를 정규식으로 파싱하여 User 객체 리스트로 반환"""
    result = []
    lines = text.strip().split('\n')
    for line in lines:
        if not line:
            continue
        # 정규식으로 다양한 구분자(|, /, ,)를 한 번에 처리
        parts = re.split(r'[,|/]', line)
        
        user_dict = {}
        for p in parts:
            k_v = p.split(":")
            if len(k_v) == 2:
                user_dict[k_v[0].strip()] = k_v[1].strip()
                
        # 딕셔너리를 객체로 변환
        if user_dict:
            result.append(User(
                name=user_dict.get('name'),
                age=int(user_dict.get('age', 0)),
                phone=user_dict.get('phone'),
                dept=user_dict.get('dept')
            ))
    return result
`
