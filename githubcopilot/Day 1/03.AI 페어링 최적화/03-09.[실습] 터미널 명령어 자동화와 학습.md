### 실습: 터미널 명령어 자동화와 학습 (Terminal Chat)

본 실습은 GitHub Copilot의 /terminal 명령어 기능을 활용하여 복잡한 운영체제 명령어를 IDE 외부(브라우저 검색 등)로 나가지 않고도 즉시 생성하고 실행하는 편리함을 체험하는 과정입니다.

---

#### 1. 준비: 실습 환경 구성

실습을 위해 현재 프로젝트 폴더에 여러 개의 더미 파일과 대상 폴더를 생성합니다. 터미널을 열고 아래 명령어를 입력하여 환경을 준비하십시오.

**터미널 입력:**
```bash
# 여러 개의 파이썬 파일과 텍스트 파일 생성
touch file1.py file2.py file3.txt script.py notes.txt
# 파일을 이동시킬 대상 폴더 생성
mkdir archive
```

---

#### 2. 과정: /terminal 명령어를 통한 자동화

수강생은 기억하기 어려운 리눅스/유닉스 조합 명령어를 Copilot에게 요청하여 터미널에 직접 반영합니다.

**단계 1: 터미널 도움말 호출**
1. 측면 채팅창(Copilot Chat)을 엽니다.
2. 입력창에 `/terminal` 슬래시 명령어를 포함하여 다음과 같이 요청합니다.
   > "/terminal 현재 폴더에서 .py 확장자를 가진 파일만 모두 찾아서 archive 폴더로 이동시키는 명령어를 알려줘."

**단계 2: 제안된 명령어 검토 및 삽입**
1. Copilot이 제안한 명령어(예: `find . -name "*.py" -exec mv {} archive/ \;` 또는 `mv *.py archive/`)와 각 옵션에 대한 설명을 읽습니다.
2. 명령어 블록 위에 나타나는 **[Insert into Terminal]** 버튼을 클릭합니다.
3. IDE 하단 터미널에 해당 명령어가 자동으로 입력되는지 확인합니다.

**단계 3: 명령어 실행 및 결과 확인**
1. 터미널에서 Enter를 눌러 명령어를 실행합니다.
2. `ls archive` 명령어를 입력하여 파이썬 파일들만 정상적으로 이동했는지 확인합니다.

---

#### 3. 결과 확인 및 논의

터미널 채팅을 활용했을 때 기존 방식과 비교하여 다음과 같은 장점을 확인합니다.

* **컨텍스트 스위칭(Context Switching) 최소화:** 구글 검색이나 Stack Overflow를 찾기 위해 브라우저로 이동하는 흐름을 끊고 코딩 환경에 집중할 수 있습니다.
* **학습 효과:** 단순히 명령어를 복사해주는 것이 아니라, 사용된 옵션(예: `-exec`, `{}`, `\;` 등)의 의미를 함께 설명해주어 터미널 사용 역량을 높여줍니다.
* **운영체제 적응성:** 동일한 요청을 하더라도 현재 사용 중인 OS(Windows PowerShell, macOS Zsh 등)에 최적화된 문법을 제안받을 수 있습니다.
