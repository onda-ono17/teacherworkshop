# GitHub Copilot 핵심 인터페이스: 인라인 채팅 (Inline Chat)

인라인 채팅은 코드 에디터의 흐름을 깨지 않고 편집 중인 위치에서 즉시 AI의 도움을 받을 수 있는 인터페이스입니다. 개발자가 작성 중인 코드 줄(Line)이나 선택한 블록에 집중하여 직관적인 수정을 가능하게 합니다.

---

## 1. 실행 방법 및 단축키
* **Windows/Linux:** `Ctrl + I`
* **macOS:** `Cmd + I`
* **작동 방식:** 특정 코드를 마우스로 블록 지정하거나, 코드를 작성하려는 빈 줄에서 단축키를 누르면 입력창이 나타납니다.

---

## 2. 주요 특징
* **맥락의 국소성 (Local Context):** 전체 프로젝트보다는 현재 선택한 코드와 그 주변 줄의 맥락에 집중하여 가장 관련성 높은 수정을 제안합니다.
* **차분 뷰(Diff View) 제공:** 제안된 코드가 기존 코드와 어떻게 다른지 빨간색(삭제)과 초록색(추가)으로 보여주며, 사용자는 이를 확인한 후 즉시 수락(Accept)하거나 거절(Discard)할 수 있습니다.
* **코드 흐름 유지:** 별도의 채팅창으로 시선을 옮길 필요가 없어 코딩의 몰입도를 유지하는 데 최적화되어 있습니다.

---

## 3. 주요 용도 (Use Cases)

**1) 변수명 및 함수명 리팩토링**
* **프롬프트:** "이 변수명을 더 직관적인 이름으로 변경해줘" 또는 "CamelCase를 snake_case로 바꿔줘"
* **효과:** 선택한 범위 내의 명명 규칙을 일관되게 수정합니다.

**2) 짧은 로직 구현 및 주석 변환**
* **프롬프트:** "# 이 리스트에서 중복을 제거하는 한 줄 코드를 작성해줘"
* **효과:** 주석 바로 아래에 최적화된 로직을 생성합니다.

**3) 코드 스타일 수정**
* **프롬프트:** "이 for 루프를 리스트 컴프리헨션 형식으로 변경해줘"
* **효과:** 로직의 결과는 유지하면서 문법적 설탕(Syntactic Sugar)을 적용하여 가독성을 높입니다.

---

## 4. 측면 채팅(Copilot Chat)과의 차이점

| 구분 | 인라인 채팅 (Inline Chat) | 측면 채팅 (Sidebar Chat) |
| :--- | :--- | :--- |
| **위치** | 코드 에디터 내 (팝업형) | IDE 측면 패널 (고정형) |
| **맥락 범위** | 현재 줄 및 선택 영역 중심 | 여러 파일 및 프로젝트 전체 맥락 |
| **주요 작업** | 즉각적인 코드 수정 및 생성 | 아키텍처 설계, 긴 설명, 디버깅 상담 |
| **결과 적용** | 에디터에 직접 반영 (Accept 필요) | 채팅창에서 코드를 복사하거나 삽입 |

---

## 5. 실습 팁
* **빈 줄에서 실행:** 아무것도 없는 줄에서 단축키를 누르고 명령을 내리면 새로운 코드를 생성합니다.
* **코드 선택 후 실행:** 특정 로직을 드래그한 뒤 실행하면 해당 로직을 '수정'하는 모드로 동작합니다.
* **연속 대화:** 인라인 채팅창 내에서도 제안된 결과가 마음에 들지 않으면 추가 명령을 입력하여 결과물을 정제(Refine)할 수 있습니다.
