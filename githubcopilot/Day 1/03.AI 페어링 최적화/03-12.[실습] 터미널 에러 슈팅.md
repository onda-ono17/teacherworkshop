# Hands on Lab: 터미널 에러 슈팅

개발 중 터미널에서 쏟아지는 수많은 에러 메시지를 복사해서 검색 포털이나 ChatGPT에 붙여넣기 하는 과정은 매우 번거롭습니다. 
이 실습에서는 `#terminalLastCommand` 변수와 `/fix` 명령어를 결합하여 에러를 원클릭에 가깝게 해결하는 방법을 배웁니다.

---

## 1. 개요
*   **목적:** 터미널 에러 로그를 Copilot에게 바로 넘겨서 원인을 분석하고 코드를 수정합니다.
*   **상황:** 파이썬 코드를 실행했으나 패키지 누락이나 타입 에러 등으로 실행이 중단된 상황.

---

## 2. 실습 세팅

1.  빈 파이썬 파일 `error_app.py`를 생성하고 아래 코드를 붙여넣습니다 (의도적인 에러 포함).

```python
import sys
import json

def parse_data():
    raw_data = '{"name": "Copilot", "type": "AI"}'
    # 의도적 에러: 문자열(raw_data)이 아닌 파일 객체를 기대하는 json.load를 사용함
    parsed = json.load(raw_data) 
    print(f"Name is {parsed['name']}")

if __name__ == "__main__":
    parse_data()
```

2. 해당 파일을 단축키 등을 이용해 터미널에서 실행합니다.
   \`python error_app.py\`

## 3. 실습 진행 (Action)

1. 터미널 창에 `AttributeError: 'str' object has no attribute 'read'` 와 같은 에러(Traceback)가 발생합니다.
2. 에러 로그를 드래그해서 복사할 필요 **없습니다.**
3. Copilot Chat(단축키 Ctrl+Cmd+i 또는 측면 챗)을 열고 구체적으로 다음 프롬프트를 입력합니다:
   > `@workspace /fix #terminalLastCommand 에서 발생한 에러를 분석하고 error_app.py를 수정해줘`
4. Copilot이 에러의 원인(`json.load`가 아닌 `json.loads`를 써야 함)을 정확히 파악하고 수정된 코드를 제안하는지 확인합니다.

---

## 4. 실무 팁 (Best Practices)
*   **npm, Docker, 빌드 에러:** 파이썬 뿐만 아니라 `npm run build` 실패 에러, Docker 컨테이너 실행 에러 등 터미널에서 발생하는 **모든 텍스트 에러**에 대해 이 기법을 동일하게 적용할 수 있습니다.
