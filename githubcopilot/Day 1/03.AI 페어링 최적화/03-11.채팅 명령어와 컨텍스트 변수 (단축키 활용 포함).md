# 채팅 명령어와 컨텍스트 변수 (`/`, `@`, `#`)

GitHub Copilot Chat은 AI에게 사용자의 의도(Intent)를 명확히 전달하고, 참조해야 할 배경 코드(Context)를 정확히 짚어주기 위한 특수 명령어 체계를 제공합니다.

---

## 1. 슬래시 명령어 (`/`) - 명시적 의도 전달
무엇을 하고 싶은지 AI에게 명확히 지시합니다. 직접 타이핑하지 않아도 되는 장점이 있습니다.

*   **/explain**: 선택된 코드나 파일이 어떻게 작동하는지 단계별로 설명합니다. (레거시 분석 시 유용)
*   **/fix**: 코드의 문법 오류, 버그, 또는 안티 패턴을 찾아내고 수정안을 제안합니다.
*   **/tests**: 현재 범위나 선택된 코드에 대한 단위 테스트 템플릿을 생성합니다.
*   **/doc**: 함수나 클래스에 대한 표준 주석(Javadoc, Docstring 등)을 작성합니다.

## 2. 참조 변수 (`#`) - 특정 컨텍스트 주입
Copilot이 "어느 파일" 또는 "어떤 에러"를 보고 판단해야 할지 강제합니다. 환각(Hallucination)을 줄이는 핵심 기술입니다.

*   **#file**: 특정 파일의 전체 코드를 컨텍스트로 불러옵니다. (예: `#file:utils.py 코드를 참고해서 이 쪽도 수정해줘`)
*   **#selection**: 현재 에디터에서 드래그(선택)한 코드 영역만 참조합니다.
*   **#terminalLastCommand**: 터미널에서 가장 마지막으로 실행한 커맨드와 그 에러 로그를 참조합니다. 가장 강력한 디버깅 도구 중 하나입니다.

## 3. 에이전트 마커 (`@`) - 전문 영역 지정
질문의 답변을 처리할 기본 AI 모델이 아닌, "특정 영역에 특화된 에이전트"를 호출합니다.

*   **@workspace**: 현재 열려있는 VS Code 워크스페이스(전체 폴더) 내의 모든 파일을 인덱싱하여 프로젝트 전체의 컨텍스트를 파악합니다. (예: `@workspace 사용자 인증 로직은 주로 어디에 모여있어?`)
*   **@vscode**: VS Code 편집기 자체의 사용법, 단축키, 확장 프로그램 설정에 대한 질문을 처리합니다.
*   **@terminal**: 터미널 명령어 생성, 쉘 스크립트 작성 방법에 대해 답변합니다.
