# 실습: Copilot 환각(Hallucination) 대응 및 에러 자동 복구 (/fix)

AI 파트너는 때로 문맥을 오해하거나 존재하지 않는 라이브러리를 호출하는 '환각'을 일으킬 수 있습니다. 본 실습은 **"AI가 만든 오류를 AI와 함께 해결하는 트러블슈팅(Troubleshooting)"** 능력을 기르는 데 목적이 있습니다.

---

## 1. 실습 목표
- Copilot이 생성한 잘못된 코드(환각)를 식별하고 명시적인 프롬프트로 수정 지시하기
- 터미널 런타임 에러를 측면 채팅(Copilot Chat)으로 가져와 원인 분석하기
- `/fix` 명령어를 활용한 빠르고 안전한 코드 오류 교정

---

## 2. 미션 1: 존재하지 않는 메서드(환각) 유도 및 교정

### 의도적 환각 유도
1. 파이썬 코드 파일을 열고 상단에 내장 모듈 `import datetime`을 선언합니다.
2. 인라인 채팅(Ctrl+I / Cmd+I)을 열고 매우 모호한 프롬프트를 입력합니다: 
   `"datetime 객체를 시간대 변환해서 DB에 넣으려고 해. 포맷을 변경하는 함수 만들어줘."`
3. Copilot이 종종 존재하지 않는 가상의 편의 메서드(예: `to_korean_time()`)나 호환되지 않는 방식을 제안하면 그대로 **Accept** 합니다.

### 교정 실습
1. 결과 코드에서 나타난 빨간 줄(Lint/Syntax Error)에 마우스를 올립니다.
2. 우클릭 후 **Copilot > Fix using Copilot**을 선택하거나, 인라인 채팅을 다시 열어 구체적으로 지시합니다.
   `/fix 이 메서드는 기본 모듈에 존재하지 않아. 내장 zoneinfo 모듈을 사용하는 방향으로 다시 작성해줘.`
3. 피드백이 반영된 정확한 코드로 교정되는 것을 확인하고 반영합니다.

---

## 3. 미션 2: 런타임 에러 분석과 터미널 연동

### 런타임 에러 발생
1. 작성된 코드에 강제로 논리/타입 에러를 유발합니다. (예: 문자열 변수와 정수를 더하는 연산을 `return` 직전에 추가)
2. 터미널(Terminal)에서 스크립트를 실행시켜 파이썬 Traceback 에러를 출력합니다.

### 트러블슈팅
1. 터미널의 에러 로그 텍스트를 드래그하여 블록 지정합니다.
2. 우클릭 후 **Copilot > Explain This**를 선택하거나, 터미널 인라인 채팅에서 `@terminal /explain`을 입력합니다.
3. Copilot이 분석한 에러의 원인(TypeError 등)을 파악합니다.
4. 측면 채팅으로 넘어가 `/fix 실행 시 발생한 터미널 에러를 바탕으로 기존 코드를 수정해줘.`라고 요청합니다.
5. 제안된 코드 블록에서 **Apply in Editor** 아이콘을 클릭하여 코드를 즉시 변경하고 재실행합니다.

---

## 4. 핵심 학습 포인트 (Takeaways)

- **"눈먼 수락(Blind Accept)의 위험성"**: 겉보기에 멀쩡한 코드도 런타임에 문제를 일으킬 수 있으므로 개발자의 검증이 필수입니다.
- **"문맥 전환 없는 디버깅"**: 구글링이나 스택오버플로우 검색창으로 이동할 필요 없이, IDE 내부에서 에러 텍스트 자체를 Context로 던져 복구 사이클을 단축합니다.
- **"프롬프트 구체성의 재확인"**: 환각의 90%는 '지시사항의 모호함'에서 비롯됩니다. 제약 조건과 사용할 라이브러리를 명확히(Specificity) 주면 환각은 크게 줄어듭니다.
