# 실습: 터미널 명령어 도움말 (/terminal)

본 실습은 GitHub Copilot의 /terminal 명령어를 활용하여 기억하기 어렵거나 복잡한 터미널 명령어(Git, Docker, NPM 등)에 대한 도움을 받고, 에디터를 벗어나지 않고도 터미널 작업을 수행하는 방법을 익히는 과정입니다.

---

## 1. 실습 개요 및 환경

개발 중 터미널에서 자주 겪는 복잡한 상황들을 설정합니다. 별도의 코드 파일은 필요하지 않으며, IDE의 측면 채팅창(Copilot Chat)과 통합 터미널을 사용합니다.

* **주요 대상:** Git 고급 명령어, Docker 컨테이너 관리, 클라우드 CLI 명령어 등.

---

## 2. 실습 진행 단계

수강생은 다음 세 가지 시나리오를 통해 /terminal 명령어의 효용성을 확인합니다.

**시나리오 1: Git 커밋 취소하기**
1. 상황: 방금 수행한 Git 커밋에 오류가 있어 취소하고 싶지만, 작업한 파일 내용은 그대로 유지(Staging 상태)하고 싶은 경우입니다.
2. 실행: Copilot Chat 창에 `/terminal 가장 최근의 git 커밋을 취소하되 변경 사항은 보존하는 명령어를 알려줘`라고 입력합니다.
3. 확인: `git reset --soft HEAD~1` 명령어와 각 옵션(--soft 등)에 대한 설명을 확인합니다.

**시나리오 2: 포트 점유 프로세스 확인 및 종료**
1. 상황: 특정 포트(예: 8080)가 이미 사용 중이라 서버가 실행되지 않는 경우입니다.
2. 실행: `/terminal 8080번 포트를 사용 중인 프로세스를 찾아서 종료하는 명령어를 알려줘`라고 입력합니다.
3. 확인: OS 환경(Windows/macOS/Linux)에 맞는 명령어(예: `lsof -i :8080` 또는 `netstat`)를 제안받습니다.

**시나리오 3: Docker 미사용 자원 정리**
1. 상황: 로컬 디스크 용량이 부족하여 사용하지 않는 Docker 이미지와 컨테이너를 일괄 삭제하고 싶습니다.
2. 실행: `/terminal 사용하지 않는 모든 docker 컨테이너와 이미지를 삭제하는 안전한 명령어를 알려줘`라고 입력합니다.
3. 확인: `docker system prune` 명령어의 작동 원리와 주의사항을 확인합니다.

---

## 3. 결과: 명령어 적용 및 실행

Copilot이 제안한 명령어 블록 위에는 **[Insert into Terminal]** 버튼 또는 **[Copy]** 아이콘이 나타납니다.

* **수행:** 제안된 명령어를 확인한 후 [Insert into Terminal]을 클릭하여 IDE 하단 터미널에 명령어가 자동으로 입력되는지 확인합니다.
* **주의:** 명령어를 실행하기 전, Copilot이 제공한 설명을 읽고 현재 의도와 일치하는지 반드시 검토합니다.
