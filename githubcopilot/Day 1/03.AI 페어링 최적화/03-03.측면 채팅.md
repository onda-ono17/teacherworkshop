# GitHub Copilot 핵심 인터페이스: 측면 채팅 (Copilot Chat)

측면 채팅은 IDE의 측면 패널에 위치한 대화형 인터페이스로, 단순한 코드 수정을 넘어 프로젝트 전체의 구조 설계, 복잡한 로직 설명, 버그 원인 분석 등 깊이 있는 상담이 필요할 때 활용하는 강력한 도구입니다.

---

## 1. 실행 방법 및 특징
* **실행:** IDE 왼쪽 또는 오른쪽의 대화 아이콘을 클릭하거나 단축키(Windows/Linux: `Ctrl + Alt + I`, macOS: `Cmd + Shift + I`)를 사용하여 호출합니다.
* **대화형 구조:** 챗봇 형태의 UI를 통해 질문과 답변을 주고받으며 결과를 점진적으로 구체화할 수 있습니다.
* **전역적 맥락(Global Context):** 현재 활성화된 파일뿐만 아니라 프로젝트 내의 여러 파일을 동시에 참조하여 답변을 생성할 수 있습니다.

---

## 2. 핵심 슬래시 명령어 (Slash Commands)

채팅창에 `/`를 입력하여 사용할 수 있는 예약어들로, 복잡한 설명 없이도 AI에게 명확한 태스크를 부여할 수 있습니다.

* **/explain:** 선택한 코드의 작동 원리를 단계별로 설명해줍니다. 초보 개발자의 학습이나 레거시 코드 파악에 유용합니다.
* **/fix:** 선택한 코드의 버그를 진단하고 수정안을 제안합니다. 런타임 에러나 논리 오류 해결에 최적화되어 있습니다.
* **/tests:** 선택한 로직에 대한 단위 테스트(Unit Test) 코드를 자동으로 생성합니다.
* **/doc:** 함수나 클래스에 대한 문서화 주석(Docstring)을 생성하여 코드의 가독성을 높입니다.
* **/new:** 새로운 프로젝트나 파일의 기본 뼈대(Scaffolding)를 설계해달라고 요청할 때 사용합니다.

---

## 3. 컨텍스트 변수 (Context Variables)

질문의 범위를 명확히 지정하여 답변의 정확도를 높이는 기능입니다.

* **@workspace:** 프로젝트 전체 구조를 바탕으로 질문합니다. "우리 프로젝트에서 인증 로직이 어디에 구현되어 있어?"와 같은 질문이 가능합니다.
* **#file:** 특정 파일을 지정하여 질문합니다. 예: "#file:auth_service.py 의 로그인 로직을 분석해줘."
* **#selection:** 현재 에디터에서 드래그하여 선택한 영역만 참고하도록 제한합니다.

---

## 4. 인라인 채팅 vs 측면 채팅 활용 전략

| 상황 | 추천 인터페이스 | 이유 |
| :--- | :--- | :--- |
| 변수명 변경, 짧은 로직 수정 | **인라인 채팅** | 코드 흐름을 깨지 않고 즉시 반영 가능 |
| 신규 기능의 아키텍처 설계 | **측면 채팅** | 넓은 맥락을 고려한 긴 설명과 대안 제시 필요 |
| 복잡한 런타임 오류 해결 | **측면 채팅** | 에러 로그를 붙여넣고 단계별 원인 분석 가능 |
| 간단한 주석 추가 | **인라인 채팅** | 특정 줄에 바로 삽입하는 것이 효율적 |
