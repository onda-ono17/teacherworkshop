# 실습: 문서화 및 주석 생성 (/doc)

본 실습은 GitHub Copilot의 슬래시 명령어인 /doc을 활용하여 복잡한 로직에 대한 설명과 표준화된 문서화 주석(Docstring)을 자동으로 생성하고, 코드의 가독성 및 유지보수성을 높이는 과정을 학습합니다.

---

## 1. 준비: 문서화가 필요한 코드 작성

주석이 전혀 없어 로직 파악이 어려운 함수를 준비합니다. 아래 함수는 주식 투자 수익률과 세금을 계산하는 로직을 담고 있습니다.

**파일명: finance_utils.py**
```python
def calculate_net_profit(buy_price, sell_price, quantity, tax_rate):
    raw_profit = (sell_price - buy_price) * quantity
    tax_amount = raw_profit * tax_rate
    
    if raw_profit > 0:
        net_profit = raw_profit - tax_amount
    else:
        net_profit = raw_profit
        
    return net_profit, tax_amount
```

---

## 2. 과정: /doc 명령어를 통한 문서화 자동화

수강생은 수동으로 작성하기 번거로운 표준 주석 형식을 Copilot을 통해 생성합니다.

**단계 1: 함수 선택 및 명령어 입력**
1. finance_utils.py 파일에서 calculate_net_profit 함수 전체를 드래그하여 선택합니다.
2. 측면 채팅창(Copilot Chat)에 /doc 명령어를 입력하고 전송합니다.

**단계 2: 생성된 문서 검토**
* 확인 포인트: Copilot이 다음과 같은 항목을 포함한 표준 주석(예: Google 스타일 또는 Sphinx 스타일)을 생성했는지 확인합니다.
    - 함수의 목적 요약
    - 인자(Args) 설명: 각 매개변수의 의미와 데이터 타입
    - 반환값(Returns) 설명: 반환되는 튜플의 구성 요소 설명

**단계 3: 코드 적용 및 인라인 확인**
1. 채팅창에 나타난 제안을 확인하고 [Insert at Cursor] 버튼을 누르거나 코드를 복사하여 함수 상단에 적용합니다.
2. 에디터에서 함수 이름 위에 마우스를 올렸을 때(Hover), 생성된 주석이 도구 설명(Tooltip)으로 깔끔하게 나타나는지 확인합니다.

## 3. 결과: 생성된 문서화 예시 (Google 스타일)

Copilot은 일반적으로 아래와 같이 정돈된 형태의 주석을 제안합니다.

```python
def calculate_net_profit(buy_price, sell_price, quantity, tax_rate):
    """
    매수/매도 가격과 수량, 세율을 바탕으로 순이익과 세금을 계산합니다.

    Args:
        buy_price (float): 주당 매수 가격
        sell_price (float): 주당 매도 가격
        quantity (int): 거래 수량
        tax_rate (float): 적용 세율 (예: 0.15)

    Returns:
        tuple: (net_profit, tax_amount) 형태의 튜플. 
               net_profit은 세금을 제외한 순이익, tax_amount는 계산된 세금.
    """
    raw_profit = (sell_price - buy_price) * quantity
    tax_amount = raw_profit * tax_rate
    
    if raw_profit > 0:
        net_profit = raw_profit - tax_amount
    else:
        net_profit = raw_profit
        
    return net_profit, tax_amount
```
