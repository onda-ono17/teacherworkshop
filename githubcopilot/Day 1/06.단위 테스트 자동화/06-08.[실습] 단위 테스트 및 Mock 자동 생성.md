# 실습: 단위 테스트 및 Mock 자동 생성

순수 비즈니스 로직이 아닌, 외부 API 요청이나 DB 처리가 포함된 실제 서버 환경의 코드를 단위 테스트하는 기법을 배웁니다.

---

## 1. 개요 (Context)
*   **상황:** 외부 API를 통해 환율(Exchange Rate) 정보를 가져와 물건 가격을 달러(USD)로 환산하는 로직이 있습니다.
*   **목표:** 외부의 실제 환율 API 서버를 찌르지(실제 통신하지) 않고, 예상된 가짜 응답(Mock)을 반환하게 설정하여 로직 자체만을 테스트하는 단위 테스트 코드를 Copilot으로 작성합니다.

---

## 2. 실습 세팅

1.  `exchange_service.py` 라는 파일을 만들고 아래 코드를 작성합니다.

```python
import requests

def get_current_exchange_rate(currency_code: str) -> float:
    # 이 API는 실제로 언제 바뀔지 모르며 네트워크 지연을 유발합니다.
    response = requests.get(f"https://api.fake-exchange.com/rates/{currency_code}")
    response.raise_for_status()
    data = response.json()
    return float(data.get("rate", 1.0))

def calculate_price_in_usd(krw_price: float) -> float:
    # 비즈니스 로직 : 외부 시스템(get_current_exchange_rate)에 직접 의존합니다.
    usd_rate = get_current_exchange_rate("USD")
    return round(krw_price / usd_rate, 2)
```

## 3. 실습 진행 (Action)

1.  작성된 코드를 전체 드래그하거나 활성화한 상태로 Copilot Chat을 엽니다.
2.  단위 테스트 생성을 지시하기 위해 다음 프롬프트를 입력합니다.
    > `/tests calculate_price_in_usd 함수에 대한 단위 테스트 코드를 만들어줘. 주의할 점은 requests 패키지의 기능을 Mocking (patch 등 활용)하여 실제 API 호출이 안 가도록 하고, USD 환율은 1300원으로 고정해서 반환하게 설정해.`
3.  **코드 검토:** Copilot이 생성한 테스트 코드에서 `unittest.mock.patch` 또는 `mocker`를 사용하여 `get_current_exchange_rate` 함수 혹은 `requests.get` 의 반환값을 임의 조작(Mocking)했는지 확인합니다.
4.  새로운 `test_exchange_service.py` 파일을 만들어 코드를 수락하고, 터미널에서 `pytest` (또는 python -m unittest) 명령어로 외부 의존성 없이도 Pass 떨어지는 것을 확인합니다.

---

## 4. 실무 팁 (Best Practices)
*   위와 같이 네트워크나 DB의 지연 시간 없이 초고속으로 수행될 수 있도록 테스트를 격리하는 연습이 백엔드/프론트엔드 실무에 필수적입니다. 데이터베이스 세션 Mocking이나 외부 파일(Local File) I/O 등에도 동일한 방식으로 "외부 환경을 격리해라"는 인스트럭션을 주시면 됩니다.
