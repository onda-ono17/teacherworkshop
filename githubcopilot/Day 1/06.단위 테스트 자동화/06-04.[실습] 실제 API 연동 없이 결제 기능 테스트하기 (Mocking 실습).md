# 실습: 실제 API 연동 없이 결제 기능 테스트하기 (Mocking 실습)

본 실습은 외부 결제 게이트웨이(Payment Gateway) API가 아직 개발되지 않았거나, 실제 결제 비용 발생을 방지해야 하는 상황을 가정합니다. Mock 객체를 활용하여 다양한 결제 시나리오를 안전하고 빠르게 검증하는 것이 목표입니다.

---

## 1. 실습 시나리오

여러분은 쇼핑몰의 주문 처리 시스템을 개발하고 있습니다. 주문 완료를 위해서는 외부 업체의 결제 API를 호출해야 합니다. 하지만 외부 API는 호출할 때마다 실제 비용이 발생하며, 네트워크 상태에 따라 응답이 불안정할 수 있습니다.

**요구사항:**
1. 결제 성공 시 주문 상태는 'COMPLETED'가 되어야 합니다.
2. 결제 거절 시 주문 상태는 'FAILED'가 되어야 하며 사용자에게 알림을 보냅니다.
3. 네트워크 타임아웃 등 외부 API 에러 발생 시 재시도 로직이 작동해야 합니다.

---

## 2. 기초 코드 (Base Code)

수강생은 아래 코드를 바탕으로 테스트 코드를 작성합니다.

**Python (FastAPI 스타일):**

class PaymentGateway:
    def process_payment(self, amount):
        # 실제로는 외부 API를 호출하는 복잡한 로직
        pass

class OrderService:
    def __init__(self, gateway: PaymentGateway):
        self.gateway = gateway

    def checkout(self, order_id, amount):
        try:
            success = self.gateway.process_payment(amount)
            if success:
                return "COMPLETED"
            return "FAILED"
        except Exception:
            return "RETRY_REQUIRED"

## 3. 실습 과제 (Tasks)

### Task 1: 결제 성공 시나리오 테스트
Copilot을 활용하여 결제 게이트웨이가 항상 True를 반환하도록 Mock을 설정하고, 주문 상태가 'COMPLETED'가 되는지 확인하는 코드를 작성하세요.

### Task 2: 결제 거절 시나리오 테스트
결제 게이트웨이가 False를 반환하도록 Mock을 설정하고, 주문 상태가 'FAILED'로 변경되는지 검증하세요.

### Task 3: 예외 상황(Timeout) 테스트
외부 API 호출 시 TimeoutError가 발생하는 상황을 Mock으로 시뮬레이션하고, 시스템이 'RETRY_REQUIRED'를 정확히 반환하는지 테스트하세요.
