# 실습: GitHub Copilot 테스트 자동화 및 Mocking 실무

본 실습은 GitHub Copilot의 강력한 테스트 생성 기능을 활용하여, 단위 테스트 작성 시간을 단축하고 Mock 객체를 이용한 정교한 검증 기법을 익히는 것을 목표로 합니다.

---

## 1. 활동 1: /tests 명령어로 기본 테스트 생성

수강생들이 가장 먼저 접하게 될 기능으로, 기존 코드에 대한 기초 테스트 수트를 순식간에 구축하는 경험을 합니다.

* **실행 방법:**
    1. 14:00 세션에서 생성한 '주문 관리 시스템'의 OrderService.py(또는 OrderService.java) 파일을 엽니다.
    2. 테스트하고자 하는 함수(예: checkout, register_member 등)를 마우스로 드래그하여 선택합니다.
    3. Copilot Chat 창을 열고 `/tests`를 입력한 후 엔터를 누릅니다.

* **학습 포인트:**
    - Copilot이 현재 프로젝트 환경에 맞는 테스트 라이브러리(pytest 또는 JUnit)를 자동으로 판단하여 임포트하는지 확인합니다.
    - 입력값에 따른 가장 기본적인 성공 케이스(Happy Path)가 어떻게 구성되는지 분석합니다.
    - 제안된 코드를 [Insert into New File] 버튼을 눌러 `tests/` 폴더에 저장하는 워크플로우를 익힙니다.

---

## 2. 활동 2: Mocking 프롬프트 작성 및 의존성 절연

데이터베이스나 외부 API와 연결된 복잡한 로직을 Mock 객체로 격리하여 테스트하는 고급 기법을 실습합니다.

* **실행 방법:**
    - 데이터베이스 조회 로직이나 외부 결제 API 호출 코드가 포함된 함수를 선택합니다.
    - Copilot Chat에 구체적인 라이브러리와 목적을 명시하여 프롬프트를 작성합니다.

* **프롬프트 예시:**
    - "이 함수는 데이터베이스에 의존하고 있어. unittest.mock을 사용해서 데이터베이스 호출 부분을 Mocking하고, 가짜 데이터를 반환하도록 테스트 코드를 작성해줘."
    - "Java의 Mockito를 활용해서 PaymentGateway의 응답을 Mocking하는 @Test 메서드를 만들어줘."

* **학습 포인트:**
    - `return_value`나 `when(...).thenReturn(...)`을 통해 외부 환경을 제어하는 법을 배웁니다.
    - 실제 인프라 구축 없이도 비즈니스 로직의 무결성을 검증할 수 있음을 체감합니다.

---

## 3. 활동 3: 에러 케이스 자동화 및 체크리스트 검증

앞선 세션에서 설계한 '에러 시나리오 체크리스트'를 바탕으로, 발생 가능한 모든 예외 상황을 일괄적으로 테스트 코드로 변환합니다.

* **실행 방법:**
    - 15:00 세션에서 작성한 '에러 시나리오 체크리스트' 텍스트를 복사합니다.
    - Copilot Chat에 체크리스트와 함께 구현 명령을 전달합니다.

* **프롬프트 예시:**
    - "아래 체크리스트에 정의된 에러 상황들을 검증하는 테스트 메서드들을 한꺼번에 추가해줘. [체크리스트 내용 붙여넣기]"
    - "구매 금액이 음수이거나 수량이 100개를 초과할 때 발생하는 ValueError 테스트 케이스를 생성해줘."

* **학습 포인트:**
    - 자연어로 작성된 요구사항 명세가 어떻게 엄격한 테스트 코드로 치환되는지 확인합니다.
    - 경계값 분석과 동등 분할 기법이 적용된 테스트 데이터가 정확히 삽입되는지 검토합니다.
