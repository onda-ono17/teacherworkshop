# 실습: 주문 결제 시스템 구축(Order management system)

본 문서는 GitHub Copilot과 협업하여 엔터프라이즈 수준의 Java 애플리케이션을 설계하고 구현하는 전 과정을 담고 있습니다. Java 21의 최신 기능과 Spring Boot 3.3의 안정성을 결합하여 고가용성 시스템을 구축해 봅니다.

---

## 1. 전역 지침 (System Instructions)

프로젝트 시작 전 Copilot에게 주입할 핵심 규칙입니다. `.github/copilot-instructions.md` 파일에 저장하거나 대화 시작 시 입력하세요.

```markdown
- **Environment**: Java 21, Spring Boot 3.3+, Maven, PostgreSQL.
- **Architecture**: Domain-Driven Design (DDD) 기반 레이어드 아키텍처.
- **Rules**:
  1. DTO는 Java `record` 타입을 사용한다.
  2. 모든 비즈니스 로직은 인터페이스 기반의 Service 레이어에 구현하며 `@Transactional`을 필수 적용한다.
  3. 동시성 제어를 위해 JPA `@Lock(LockModeType.PESSIMISTIC_WRITE)`를 사용한다.
  4. 예외 처리는 `@RestControllerAdvice`를 통해 공통 형식으로 응답한다.
```

---

## 2. 단계별 실습 및 프롬프트 구성

### 1단계: 프로젝트 스캐폴딩 (Scaffolding)
**프롬프트:**
> "Java 21, Spring Boot 3.3, JPA, PostgreSQL을 사용하는 주문 시스템의 초기 구조를 잡아줘. domain, dto, controller, service, repository 패키지를 포함하고, Order, Product, Stock 엔티티를 설계해줘. 또한 DB 연동을 위한 docker-compose.yml과 application.yml 설정을 작성해줘."

**핵심 결과물 (docker-compose.yml):**
```yaml
services:
  db:
    image: postgres:16-alpine
    container_name: order-db
    ports: ["5432:5432"]
    environment:
      POSTGRES_DB: order_db
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
```

---

### 2단계: 복합 비즈니스 로직 구현 (Implementation)
**프롬프트:**
> "시니어 아키텍트 관점에서 OrderService를 구현해줘. 주문 시 StockRepository에서 재고를 조회할 때 비관적 락(PESSIMISTIC_WRITE)을 걸어 동시성 이슈를 차단해야 해. 재고가 부족하면 OutOfStockException을 던지고, 충분하면 재고 차감 후 주문을 저장하는 로직을 @Transactional 내에 구현해줘."

---

### 3단계: 테스트 전략 및 단위 테스트 (Testing)
**프롬프트:**
> "QA 전문가로서 placeOrder 메서드에 대해 경계값 분석(BVA) 테스트 시나리오를 작성해줘. 이후 JUnit 5와 Mockito를 사용하여, 재고가 1개일 때 성공하고 0개일 때 실패하는 케이스를 포함한 단위 테스트 코드를 작성해줘."

**테스트 시나리오 예시:**
| 테스트 케이스 | 입력 값 (주문 수량) | 상태 (재고) | 기대 결과 |
| :--- | :--- | :--- | :--- |
| **정상 주문** | 1 | 10 | 성공 (재고 9) |
| **경계값 성공** | 1 | 1 | 성공 (재고 0) |
| **경계값 실패** | 1 | 0 | OutOfStockException |

---

### 4단계: 간단한 웹 UI 구성 (Frontend)
**프롬프트:**
> "수강생들이 실제 동작을 확인할 수 있도록 Thymeleaf 기반의 상품 목록 및 주문 페이지(index.html)를 작성해줘. '주문하기' 버튼 클릭 시 Fetch API를 통해 백엔드 /api/orders를 호출하고 성공/실패 메시지를 얼럿으로 띄우는 코드를 포함해줘."

## 3. 환경 설정 및 체크리스트

### 필수 환경 설정 (application.yml)
```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/order_db
    username: user
    password: password
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true
    properties:
      hibernate:
        format_sql: true
```
