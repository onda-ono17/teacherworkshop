### 주요 단위 테스트 프레임워크 핵심 요약

단위 테스트를 효율적으로 작성하고 관리하기 위해서는 각 프로그래밍 언어의 대표적인 테스트 프레임워크를 숙달해야 합니다. 파이썬의 Pytest와 자바의 JUnit5의 핵심 기능을 정리했습니다.

---

#### 1. Pytest (Python)

파이썬 환경에서 가장 널리 사용되는 테스트 프레임워크로, 간결한 문법과 강력한 확장성이 특징입니다.

* **단순한 assert 문 사용**
    - 별도의 테스트 API를 배울 필요 없이 파이썬 표준 `assert` 문을 그대로 사용합니다.
    - 테스트 실패 시 비교 대상의 값을 상세히 보여주는 풍부한 오류 리포트를 제공합니다.

* **Fixtures (피쳐)**
    - 테스트 전후의 상태를 관리하기 위한 기능입니다. 데이터베이스 연결, 임시 파일 생성 등 테스트 준비(Setup)와 정리(Teardown)를 모듈화하여 재사용할 수 있습니다.
    - `@pytest.fixture` 데코레이터를 사용하여 정의하며, 테스트 함수의 인자로 이름을 넘겨주는 것만으로 의존성 주입이 가능합니다.



* **파라미터화 (@pytest.mark.parametrize)**
    - 동일한 로직에 대해 여러 벌의 입력값과 기대 결과값을 테스트해야 할 때 유용합니다.
    - 코드 중복 없이 하나의 테스트 함수로 수많은 데이터 셋을 검증할 수 있어 경계값 테스트 등에 최적화되어 있습니다.

---

#### 2. JUnit5 (Java)

자바 생태계의 표준 테스트 프레임워크로, 세분화된 모듈 구조와 강력한 어노테이션 시스템을 제공합니다.

* **어노테이션 기반 제어**
    - 테스트의 생명주기를 직관적으로 관리할 수 있는 다양한 어노테이션을 지원합니다.
    - `@Test`: 해당 메서드가 테스트 메서드임을 명시합니다.
    - `@BeforeEach` / `@AfterEach`: 각 테스트 실행 전후에 수행할 작업을 지정합니다.
    - `@DisplayName`: 테스트 결과 리포트에 표시될 이름을 자연어로 지정할 수 있습니다.



* **단언문 (Assertions)**
    - `org.junit.jupiter.api.Assertions` 클래스에서 제공하는 다양한 정적 메서드를 사용합니다.
    - `assertEquals(expected, actual)`: 예상 값과 실제 값이 일치하는지 확인합니다.
    - `assertThrows(exception, executable)`: 특정 코드 실행 시 기대하는 예외가 발생하는지 검증합니다.

---

#### 3. 프레임워크별 주요 특징 비교

| 구분 | Pytest (Python) | JUnit5 (Java) |
| :--- | :--- | :--- |
| **검증 방식** | 표준 assert 문 활용 | Assertions 클래스의 메서드 활용 |
| **환경 설정** | Fixture (의존성 주입 방식) | @Before / @After (어노테이션 방식) |
| **가독성** | 파이썬다운 간결함 강조 | 정적인 타입과 명확한 구조 강조 |
| **데이터 기반 테스트** | @pytest.mark.parametrize | @ParameterizedTest |
