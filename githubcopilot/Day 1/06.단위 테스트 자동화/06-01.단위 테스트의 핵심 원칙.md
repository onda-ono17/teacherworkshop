### 단위 테스트의 핵심 원칙: FIRST

성공적인 소프트웨어 개발과 유지보수를 위해서는 고품질의 단위 테스트가 필수적입니다. 좋은 단위 테스트를 작성하기 위해 전 세계적으로 통용되는 다섯 가지 핵심 원칙인 FIRST를 준수해야 합니다.

---

#### 1. Fast (빠르게)
* **내용:** 테스트는 매우 빠르게 실행되어야 합니다.
* **이유:** 테스트 실행이 느리면 개발자가 테스트를 자주 돌리는 것을 꺼리게 되고, 이는 결국 코드 품질 저하로 이어집니다. 수백, 수천 개의 테스트가 단 몇 초 내에 완료되어야 합니다.

#### 2. Independent (독립적으로)
* **내용:** 각 테스트 케이스는 서로 의존해서는 안 됩니다.
* **이유:** 특정 테스트의 성공이나 실패가 다른 테스트에 영향을 주어서는 안 됩니다. 테스트는 어떤 순서로 실행되어도 결과가 동일해야 하며, 실패 시 원인을 정확히 파악할 수 있도록 격리(Isolated)되어야 합니다.

#### 3. Repeatable (반복 가능하게)
* **내용:** 어떤 환경에서도 동일한 결과가 반복되어야 합니다.
* **이유:** 개발자의 로컬 PC, 빌드 서버(CI), 혹은 네트워크가 없는 환경에서도 결과가 같아야 합니다. 환경에 따라 결과가 달라지는 테스트는 신뢰할 수 없습니다.

#### 4. Self-Validating (자가 검증하는)
* **내용:** 테스트는 자체적으로 성공 또는 실패를 판별해야 합니다.
* **이유:** 테스트 결과를 확인하기 위해 개발자가 로그 파일을 직접 열어보거나 수동으로 데이터를 비교해서는 안 됩니다. 테스트 통과 여부는 "Pass" 또는 "Fail"이라는 명확한 결과로 즉시 나타나야 합니다.

#### 5. Timely (적시에)
* **내용:** 테스트는 적절한 시기에 작성되어야 합니다.
* **이유:** 가장 이상적인 시기는 실제 코드를 구현하기 직전(TDD)입니다. 기능을 모두 구현한 뒤에 테스트를 작성하려고 하면 코드가 테스트하기 어렵게 설계되었음을 깨닫거나, 테스트 작성을 뒤로 미루게 될 가능성이 큽니다.

### 주요 단위 테스트 프레임워크 핵심 요약

언어별로 가장 널리 사용되는 프레임워크의 특징을 이해하고 프로젝트 성격에 맞는 도구를 선택해야 합니다.

#### 1. Pytest (Python)
파이썬의 표준 unittest보다 간결하고 확장성이 뛰어난 프레임워크입니다.
* **단순한 assert:** 복잡한 API 대신 파이썬 기본 `assert` 문을 그대로 사용하여 가독성이 높습니다.
* **Fixtures:** 테스트에 필요한 사전 데이터나 환경 설정을 관리하는 강력한 기능을 제공합니다. 재사용성이 높고 의존성 주입이 용이합니다.
* **Parametrize:** `@pytest.mark.parametrize` 어노테이션을 사용하여 하나의 테스트 함수에 여러 데이터 세트를 입력하여 반복 검증할 수 있습니다.

#### 2. JUnit5 (Java)
자바 생태계에서 가장 강력하고 표준화된 테스트 프레임워크입니다.
* **어노테이션 기반:** `@Test` (테스트 메서드), `@BeforeEach` (사전 작업), `@DisplayName` (테스트 이름 지정) 등 직관적인 어노테이션을 제공합니다.
* **Assertions:** `Assertions.assertEquals(expected, actual)`와 같은 정적 메서드를 통해 결과 값을 엄격하게 검증합니다.
* **모듈화:** JUnit Platform, JUnit Jupiter, JUnit Vintage로 나뉘어 다양한 실행 환경과 하위 호환성을 지원합니다.
