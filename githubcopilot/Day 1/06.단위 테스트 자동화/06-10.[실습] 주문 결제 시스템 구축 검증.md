# [실습] 주문 결제 시스템 구축 검증

본 문서는 GitHub Copilot과 협업하여 엔터프라이즈 수준의 Java 애플리케이션을 설계하고, 설정한 규칙이 제대로 준수되는지 검증하는 전 과정을 담고 있습니다.

---

## 1. 전역 지침 (System Instructions)

프로젝트 루트의 `.github/copilot-instructions.md` 파일에 저장하거나 대화 시작 시 입력하여 Copilot의 페르소나를 설정합니다.

```markdown
- **Environment**: Java 21, Spring Boot 3.3+, Maven, PostgreSQL.
- **Architecture**: Domain-Driven Design (DDD) 기반 레이어드 아키텍처.
- **Rules**:
  1. DTO는 Java `record` 타입을 사용한다.
  2. 모든 비즈니스 로직은 인터페이스 기반의 Service 레이어에 구현하며 `@Transactional`을 필수 적용한다.
  3. 동시성 제어를 위해 JPA `@Lock(LockModeType.PESSIMISTIC_WRITE)`를 사용한다.
  4. 예외 처리는 `@RestControllerAdvice`를 통해 공통 형식으로 응답한다.
```

---

## 2. 단계별 실습 시나리오

### 1단계: 프로젝트 스캐폴딩 (Scaffolding)
**프롬프트:**
> "Java 21, Spring Boot 3.3, JPA, PostgreSQL을 사용하는 주문 시스템의 초기 구조를 잡아줘. domain, dto, controller, service, repository 패키지를 포함하고, Order, Product, Stock 엔티티를 설계해줘. 또한 DB 연동을 위한 docker-compose.yml 설정을 작성해줘."

### 2단계: 복합 비즈니스 로직 구현 (Implementation)
**프롬프트:**
> "시니어 아키텍트 관점에서 OrderService를 구현해줘. 주문 시 StockRepository에서 재고를 조회할 때 비관적 락(PESSIMISTIC_WRITE)을 걸어야 해. 재고 부족 시 OutOfStockException을 던지고, 충분하면 재고 차감 후 주문을 저장하는 로직을 @Transactional 내에 구현해줘."

### 3단계: 테스트 전략 및 단위 테스트 (Testing)
**프롬프트:**
> "QA 전문가로서 placeOrder 메서드에 대해 경계값 분석(BVA) 테스트 시나리오를 작성해줘. 이후 JUnit 5와 Mockito를 사용하여, 재고가 1개일 때 성공하고 0개일 때 실패하는 케이스를 포함한 단위 테스트 코드를 작성해줘."

## 3. 지침 준수 검증용 프롬프트 (MCT 시연용)

지침이 제대로 적용되었는지 확인하기 위해 수강생들 앞에서 다음 프롬프트를 순서대로 입력합니다.

### 검증 1: 규칙 인지 여부 확인 (직접 질문형)
* **프롬프트:** "이 프로젝트의 핵심 코딩 규칙과 아키텍처 가이드라인을 요약해줘."
* **기대 답변:** "Java 21", "record 사용", "비관적 락", "Layered 아키텍처" 등 지침 파일의 핵심 내용 언급.

### 검증 2: 규약 준수 테스트 (코드 생성형)
* **프롬프트:** "사용자 정보를 담는 UserDTO 클래스를 만들어줘. 필드는 id, name, email이 필요해."
* **성공 케이스:** `public record UserDTO(Long id, String name, String email) { }` 형태로 생성.
* **실패 케이스:** 일반적인 `public class` 형태로 생성 시 지침 미준수.

### 검증 3: 설계 원칙 테스트 (설계 제안형)
* **프롬프트:** "재고 수량을 안전하게 변경하는 기능을 추가하고 싶어. 어떤 계층에 어떤 방식으로 구현하는 게 이 프로젝트 규칙에 맞아?"
* **기대 답변:** "Service 레이어에 @Transactional 적용", "Repository에서 PESSIMISTIC_WRITE 사용" 제안.

    show-sql: true
