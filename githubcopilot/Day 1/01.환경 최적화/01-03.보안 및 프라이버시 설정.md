# GitHub Copilot 보안 및 프라이버시 설정 가이드

기업 환경에서 GitHub Copilot을 도입할 때 가장 중요하게 검토되는 항목은 지적 재산권(IP) 보호와 데이터 보안입니다. 본 가이드는 핵심 설정인 공용 코드 매칭과 텔레메트리 데이터 처리에 대해 설명합니다.

---

## 1. Suggestions Matching Public Code (공용 코드 일치 제안 필터링)

이 설정은 GitHub Copilot이 생성한 코드 제안이 GitHub의 공개 저장소(Public Repository)에 있는 기존 코드와 약 150자 이상 일치하는지 여부를 확인하는 기능입니다.

### 설정 옵션
* **Allowed (허용):** 공개 코드와 일치하는 제안을 제한 없이 사용자에게 표시합니다. 제안된 코드의 출처(저장소 정보 및 라이선스)를 확인하고 사용할 수 있습니다.
* **Blocked (차단):** 공개 코드와 일치하는 코드가 발견될 경우, 해당 제안을 사전에 필터링하여 사용자에게 보여주지 않습니다.

### 기업 강의 핵심 포인트
* **저작권 리스크 관리:** 기업에서는 오픈소스 라이선스(GPL 등) 준수 및 지적 재산권 분쟁 방지를 위해 이 옵션을 **'Blocked'**로 설정할 것을 강력히 권장합니다.
* **조직 수준의 제어:** 개인 계정과 달리, 기업용(Business/Enterprise) 라이선스는 조직 관리자가 이 설정을 모든 구성원에게 강제로 적용(Enforce)하여 개별 변경을 제한할 수 있습니다.

---

## 2. Telemetry 및 데이터 수집 (Data Collection and Privacy)

GitHub Copilot은 서비스 개선과 품질 측정을 위해 사용자로부터 다양한 데이터를 수집합니다. 데이터의 종류에 따라 처리 방식이 상이하므로 이를 구분하여 이해해야 합니다.

### 데이터 수집 유형
1.  **사용자 참여 데이터 (User Engagement Data):** 사용자가 어떤 제안을 수락하거나 무시했는지, IDE의 성능 지표 등 사용 방식에 대한 통계 정보입니다.
2.  **코드 스니펫 (Code Snippets):** 작성 중인 코드의 맥락(Context) 정보입니다.

### 라이선스별 정책 차이
* **개인용 (Individual):** 사용자가 설정에서 데이터 수집 및 모델 학습 활용 여부를 직접 선택할 수 있습니다.
* **기업용 (Business/Enterprise):** * **코드 스니펫 비저장:** GitHub 및 OpenAI는 기업 사용자의 코드 스니펫을 모델 학습에 사용하지 않으며, 결과 반환 후 즉시 삭제합니다.
    * **투명성:** 수집된 텔레메트리 데이터는 서비스 유지 및 성능 모니터링 목적으로만 제한적으로 활용됩니다.

## 3. 프로젝트 레벨 보안: `.copilotignore`

조직의 보안 정책을 개별 개발 환경에 강제하기 위해 `.copilotignore` 파일을 사용합니다. 이는 Git의 `.gitignore`와 유사하지만, GitHub Copilot이 특정 파일이나 폴더의 내용을 **'문맥(Context)으로 읽어가는 것'을 원천 차단**하는 역할을 합니다.

### 설정 방법
1. 프로젝트 루트 경로에 `.copilotignore` (또는 `.github/copilotignore`) 파일을 생성합니다.
2. Copilot이 접근해서는 안 되는 민감한 파일 경로를 작성합니다.

### 작성 예시
```text
# 환경 변수 및 시크릿 키 차단
.env
*.pem
secrets/

# 고객 개인정보 더미 데이터 차단
data/customer_info.csv

# 특정 확장자나 빌드 결과물 차단
*.log
```

### 핵심 포인트
- "우리 회사의 소스코드가 유출될까 걱정됩니다."에 대한 기술적 방어책의 핵심입니다.
- `.env` 등은 `.gitignore`에 등록되어 있어도 개발자가 에디터에서 열어두면 Copilot이 컨텍스트로 읽어갈 위험이 있습니다. **따라서 `.copilotignore` 설정은 엔터프라이즈 환경에서 선택이 아닌 필수입니다.**

---

## 4. 기업 보안 정책 적용 가이드라인

성공적인 전사 도입을 위해 관리자는 다음과 같은 설정을 권장합니다.

1.  **Public Code Match Filter:** 기업의 법률 검토 결과에 따라 'Blocked'로 설정하여 잠재적인 라이선스 침해 가능성을 차단합니다.
2.  **Data Policy:** 기업용 라이선스 계약서(DPA)를 통해 코드가 모델 학습에 재사용되지 않음을 구성원에게 공지하여 보안 우려를 해소합니다.
3.  **IDE Extension Update:** 최신 보안 패치와 필터링 알고리즘 적용을 위해 모든 개발자가 최신 버전의 확장 프로그램을 사용하도록 관리합니다.
