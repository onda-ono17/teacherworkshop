# 실습: GitHub Copilot 환경 최적화 및 설정 점검

본 실습은 GitHub Copilot의 설치 상태를 확인하고, 개발 생산성을 극대화하기 위한 IDE 최적화 설정을 직접 적용해 보는 과정입니다.

---

## 1. 실습 1: 환경 점검 "Hello Copilot" (맥락 읽기 테스트)

설치가 완료된 후, Copilot이 단순히 텍스트를 완성하는 것을 넘어 작성 중인 코드의 맥락(Context)을 정확히 이해하고 있는지 확인합니다.

**수행 단계**
1. **파일 생성:** test_copilot.py (또는 본인이 선호하는 언어의 파일)를 생성합니다.
2. **주석 입력:** 파일 최상단에 아래 주석을 입력하고 Enter를 누릅니다.
   ```python
   # 현재 날짜와 시간을 ISO 8601 형식으로 출력하는 함수 작성
   ```
3. **제안 확인:** Copilot이 함수 정의(def get_iso_datetime():)를 제안하는지 확인합니다. Tab 키를 눌러 수락합니다.
4. **연속 제안 테스트:** 함수 내부에서 return 키워드 뒤에 올 적절한 라이브러리 활용 코드가 제안되는지 확인합니다.

## 2. 실습 2: 단축키 치트 시트 (Shortcut Cheat Sheet)

마우스 사용을 줄이고 키보드만으로 Copilot과 소통하는 연습을 합니다. 아래 표를 참조하여 각 기능을 직접 실행해 보십시오.

| 기능 | VS Code (Win/Mac) | JetBrains (Win/Mac) |
| :--- | :--- | :--- |
| 제안 수락 | Tab | Tab |
| 다음 제안 보기 | Alt + ] / Option + ] | Alt + ] / Option + ] |
| 이전 제안 보기 | Alt + [ / Option + [ | Alt + [ / Option + [ |
| 제안 거절 (숨기기) | Esc | Esc |
| 전용 패널 열기 | Ctrl + Enter | 별도 도구 창 존재 |
| 인라인 채팅 실행 | Ctrl + I / Cmd + I | Ctrl + \ / Cmd + \ |

---

## 3. 실습 3: 단축키 챌린지 (The Shortcut Challenge)

**미션:** 마우스를 전혀 사용하지 않고, 3가지 다른 방식의 '소수 구하기(Prime Number)' 로직을 제안받아 본문에 삽입하십시오.

**수행 가이드**
1. **첫 번째 제안:** `# 방법 1: 일반적인 for문을 사용한 소수 판별` 입력 후 제안 수락.
2. **두 번째 제안:** `# 방법 2: 리스트 컴프리헨션을 활용한 소수 추출` 입력 후 제안 수락.
3. **세 번째 제안:** Ctrl + Enter(또는 패널 열기)를 사용하여 표시된 여러 가지 구현 방식 중 하나를 선택하여 삽입.

---

## 프로 팁 (Pro Tip)

* **제안이 나타나지 않을 때:** Alt + \ (VS Code 기준)를 누르면 수동으로 제안을 트리거할 수 있습니다.
* **간섭 제어:** JetBrains 사용자의 경우 자동 완성 팝업이 Copilot 제안을 가린다면 Esc를 한 번 눌러 팝업만 닫으십시오. 숨겨져 있던 Copilot의 인라인 제안이 나타납니다.
