# IDE별 GitHub Copilot 최적 설정 가이드

GitHub Copilot은 IDE의 기본 설정과 조화를 이룰 때 가장 강력한 성능을 발휘합니다. 각 에디터별 핵심 설정 항목을 확인하여 최적의 개발 환경을 구축하십시오.

---

## 1. Visual Studio Code (VS Code)

VS Code는 Copilot이 가장 먼저 지원된 환경으로, 인라인 제안 기능을 활용한 설정이 핵심입니다.

### Inline Suggest 활성화 확인
Copilot의 코드 제안이 회색 텍스트(Ghost Text)로 나타나지 않는다면 아래 설정을 점검해야 합니다.
* **설정 경로:** [File] -> [Preferences] -> [Settings] (단축키: `Ctrl + ,`)
* **항목명:** `Editor: Inline Suggest: Enabled`
* **설명:** 이 항목이 체크되어 있어야 코딩 중 실시간으로 Copilot의 제안을 확인할 수 있습니다.

### Copilot 전용 패널 위치 조정
복잡한 로직에 대해 여러 개의 코드 제안을 한꺼번에 보고 싶을 때 사용하는 패널(단축키: `Ctrl + Enter`)의 위치를 조정할 수 있습니다.
* **패널 활용:** 기본적으로 새 탭이나 하단 패널에 열리지만, 마우스 우클릭 후 'Move into Side Bar'를 선택하여 측면 바에 배치하면 코드 작성 화면을 가리지 않고 제안을 실시간으로 참고할 수 있어 효율적입니다.

## 2. JetBrains IDE (IntelliJ, PyCharm, WebStorm 등)

JetBrains 계열 IDE는 강력한 자체 완성 기능(IntelliJ Completion)을 가지고 있어, Copilot 제안과의 간섭을 줄이는 것이 핵심입니다.

### 인라인 힌트(Inlay Hints) 설정
Copilot의 제안이 IDE의 기본 텍스트와 겹쳐 보이지 않도록 시각적 설정을 조정합니다.
* **설정 경로:** [Settings/Preferences] -> [Editor] -> [General] -> [Appearance]
* **항목명:** `Show inline hints`
* **설명:** Copilot의 제안이 인라인 힌트 형태로 나타나므로, 이 설정이 활성화되어 있어야 합니다. 제안된 코드가 너무 밝거나 어두워 가독성이 떨어질 경우 에디터 테마 색상에서 해당 항목의 색상을 조정할 수 있습니다.

### Completion 팝업과의 간섭 조절
IDE 자체 자동 완성 팝업과 Copilot의 제안이 동시에 나타나 혼란을 주는 상황을 관리해야 합니다.
* **설정 경로:** [Settings/Preferences] -> [Languages & Frameworks] -> [GitHub Copilot]
* **주요 설정:** `IDE 내부 자동 완성 팝업이 활성화되어 있을 때 Copilot 제안 숨기기` 옵션을 검토합니다.
* **해결 방안:** 1. 자동 완성 팝업이 띄워진 상태에서 `Esc`를 눌러 팝업만 닫으면 숨겨져 있던 Copilot의 인라인 제안이 나타납니다.
    2. 또는 Copilot 제안을 먼저 수락(Tab)하도록 우선순위를 익히는 것이 생산성 향상에 도움이 됩니다.

## 3. 요약 및 권장 사항

| 설정 항목 | VS Code 권장 설정 | JetBrains 권장 설정 |
| :--- | :--- | :--- |
| **제안 방식** | Inline Suggest 활성화 필수 | Inlay Hints 설정 확인 |
| **패널 관리** | 사이드바(Side Bar) 배치 권장 | 분할 창(Split Screen) 활용 권장 |
| **간섭 방지** | 기본 제안과 Copilot 제안 구분 | 자동 완성 팝업(`Ctrl+Space`)과 구분 사용 |
