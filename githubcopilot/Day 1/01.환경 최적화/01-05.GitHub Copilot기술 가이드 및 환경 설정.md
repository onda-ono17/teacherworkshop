# GitHub Copilot 기술 가이드 및 환경 설정

GitHub Copilot의 기술적 구동 원리, 라이선스 관리, 보안 정책 및 IDE 최적화 설정을 다룹니다.

---

## 1. GitHub Copilot 작동 원리 (Workflow)

Copilot은 단순한 코드 완성을 넘어 사용자의 개발 환경을 이해하고 도구와 상호작용하는 에이전트 방식으로 동작합니다.

### 1.1. 작동 프로세스
1. **컨텍스트 수집:** IDE 확장이 현재 파일, 인접 탭, 프로젝트 구조 및 OS 환경 정보를 수집합니다.
2. **프롬프트 생성 및 전송:** 수집된 데이터를 암호화하여 GitHub 서버로 전송합니다.
3. **LLM 추론:** OpenAI의 Codex 기반 대규모 언어 모델이 최적의 코드 뭉치를 생성합니다.
4. **도구 호출(Loop):** 에이전트형 기능의 경우, 필요에 따라 파일 읽기, 수정, 터미널 실행 등 도구를 직접 사용하여 문제를 해결합니다.
5. **필터링 및 반환:** 보안 및 저작권 검사를 거친 최종 제안을 IDE에 표시합니다.

---

## 2. 인증 및 라이선스 관리

사용자 계정 유형에 따라 인증 경로와 적용되는 정책이 상이합니다.

### 2.1. 계정 유형별 비교
* **개인용 (Individual):** 개인 계정 기반, 표준 OAuth 인증, 직접 구독 관리.
* **기업용 (Business/Enterprise):** 조직(Org) 계정 기반, **SSO(SAML) 인증** 필수, 관리자에 의한 라이선스 할당.

### 2.2. 인증 문제 해결 (Troubleshooting)
* **로그인 초기화:** IDE 하단 Accounts 아이콘을 통한 로그아웃 후 재로그인 수행.
* **네트워크 점검:** 기업 방화벽에서 `github.com` 및 `copilot-proxy.githubusercontent.com` 도메인 허용 여부 확인.
* **업데이트:** 최신 인증 프로토콜 지원을 위해 IDE 확장 프로그램을 항상 최신 버전으로 유지.

---

## 3. 보안 및 프라이버시 설정

기업 보안 담당자가 반드시 확인해야 하는 핵심 설정 항목입니다.

### 3.1. Suggestions Matching Public Code (공용 코드 일치 제안)
* **Blocked (권장):** 공개 저장소의 코드와 150자 이상 일치할 경우 제안을 차단하여 오픈소스 라이선스 리스크를 예방합니다.
* **Allowed:** 출처 확인 후 공개 코드를 제안에 포함합니다.

### 3.2. Telemetry 및 데이터 정책
* **데이터 보호:** 기업용 라이선스 환경에서 사용자의 코드 스니펫은 모델 학습에 재사용되지 않으며 반환 후 즉시 삭제됩니다.
* **통계 데이터:** 서비스 품질 개선을 위한 수락/거절 비율 등의 익명화된 데이터만 수집됩니다.

## 4. IDE별 최적 설정

개발 생산성을 높이기 위한 환경별 설정 가이드입니다.

### 4.1. Visual Studio Code
* **Inline Suggest:** `Editor: Inline Suggest: Enabled` 항목을 활성화하여 고스트 텍스트 형태의 제안을 확인합니다.
* **패널 배치:** Copilot 전용 패널(`Ctrl + Enter`)을 사이드바(Side Bar)로 이동시켜 넓은 편집 공간을 확보합니다.

### 4.2. JetBrains IDE (IntelliJ 등)
* **Inlay Hints:** `Show inline hints` 설정을 통해 제안된 코드가 시각적으로 명확히 표시되도록 조정합니다.
* **간섭 조절:** IDE 자체 자동 완성 팝업이 활성화된 상태에서 Copilot 제안이 가려질 경우 `Esc` 키를 사용하여 우선순위를 관리합니다.
