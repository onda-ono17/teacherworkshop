# GitHub Copilot 작동 워크플로우

1. **컨텍스트 수집 (Context Collection)**
   IDE(VS Code, Visual Studio 등)에 설치된 Copilot 확장이 현재 작업 중인 파일의 코드뿐만 아니라, 열려 있는 다른 탭의 파일, 커서 주변의 코드, 프로젝트 구조 등의 정보를 수집합니다. 이를 통해 현재 작성 중인 기능에 가장 적합한 맥락을 파악합니다.

2. **프롬프트 생성 및 전송 (Prompt Crafting & Transmission)**
   수집된 정보는 '프롬프트(Prompt)'라는 형태로 가공됩니다. 이 프롬프트는 암호화된 HTTPS 연결을 통해 GitHub의 Copilot 서비스로 안전하게 전송됩니다. 이때 개인 정보나 민감한 데이터가 포함되지 않도록 필터링 과정이 포함될 수 있습니다.

3. **LLM 추론 (Inference by Large Language Model)**
   전송된 프롬프트는 GitHub 서버에 있는 대규모 언어 모델(주로 OpenAI의 Codex 계열 모델)로 전달됩니다. 모델은 수억 줄의 공개 코드를 학습한 데이터를 바탕으로, 주어진 맥락 뒤에 올 가장 확률이 높은 코드 뭉치를 생성합니다.

4. **필터링 및 반환 (Filtering & Return)**
   생성된 코드 제안은 사용자에게 전달되기 전, 보안 취약점이나 중복 코드(Public Code Match) 여부 등을 확인하는 필터링 과정을 거칩니다. 검증이 완료된 제안은 다시 IDE로 전송됩니다.

5. **최종 제안 및 선택 (Suggestion & Acceptance)**
   IDE의 편집기 창에 회색 글씨(Ghost Text) 형태로 코드가 나타나며, 개발자는 Tab 키를 눌러 이를 수락하거나 무시하고 직접 코딩을 계속할 수 있습니다.

# 어떤 맥락(Context)을 바탕으로 답변하나요?

Copilot은 단순히 현재 커서가 위치한 줄만 보지 않습니다. 다음과 같은 요소들을 종합적으로 고려하여 "Neighboring Tabs" 알고리즘 등을 통해 최적의 답변을 내놓습니다.

* **현재 파일의 상하단 코드:** 함수의 정의, 변수명, 사용 중인 라이브러리.
* **열려 있는 다른 탭의 내용:** 현재 파일에서 참조하고 있는 클래스나 유틸리티 함수의 내용.
* **파일 경로 및 프로젝트 구조:** 현재 파일이 어떤 폴더(예: `controllers`, `models`)에 있는지에 따른 코딩 패턴.
* **주석 및 문서화 주석:** 개발자가 작성한 자연어 설명을 해석하여 로직으로 변환.
