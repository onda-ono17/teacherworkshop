# 표준 프로젝트 구조의 중요성: 협업과 AI를 위한 약속

본 세션에서는 프로그래밍 언어별 표준 프로젝트 구조가 유지보수에 미치는 영향과, 특히 GitHub Copilot과 같은 AI 도구가 이 구조를 어떻게 활용하여 더 정확한 코드를 제안하는지 학습합니다.

---

## 1. 협업의 약속: 왜 표준 구조인가?

프로젝트 구조는 개발자 간의 소리 없는 대화입니다. Python의 Poetry나 Java의 Maven/Gradle과 같은 도구들이 제안하는 표준 구조를 따르는 것은 다음과 같은 실무적 이점을 제공합니다.

* **예측 가능성 (Predictability):** 새로운 개발자가 프로젝트에 합류했을 때, 설정 파일은 어디에 있는지, 비즈니스 로직과 테스트 코드는 각각 어디에 위치하는지 찾지 않아도 알 수 있습니다.
* **도구 체인과의 호환성:** CI/CD 파이프라인, 정적 분석 도구, 빌드 도구들은 특정 폴더 구조를 전제로 작동합니다. 표준을 지키면 추가 설정 없이 이러한 도구들을 즉시 사용할 수 있습니다.
* **유지보수의 용이성:** 프로젝트가 커짐에 따라 파일이 늘어나도 계층화된 표준 구조 덕분에 코드의 응집도는 높이고 결합도는 낮추는 설계가 가능해집니다.

| 언어 | 표준 빌드/관리 도구 | 표준 구조 예시 |
| :--- | :--- | :--- |
| **Python** | Poetry, Pyenv | `src/`, `tests/`, `pyproject.toml` |
| **Java** | Maven, Gradle | `src/main/java/`, `src/test/java/`, `pom.xml` |
| **JavaScript** | NPM, Yarn, PNPM | `src/`, `public/`, `package.json` |

---

## 2. GitHub Copilot과 구조: AI의 맥락 파악 방식

GitHub Copilot은 단순히 텍스트를 생성하는 것이 아니라, 프로젝트의 전체적인 '맥락(Context)'을 읽어 들입니다. 이때 **폴더 구조와 파일명**은 AI가 맥락을 파악하는 핵심 이정표가 됩니다.



### 2.1. AI가 구조를 인식하는 법
* **경로 기반 추론:** Copilot은 현재 편집 중인 파일의 경로를 확인합니다. 예를 들어 파일이 `tests/` 폴더 아래에 있다면, AI는 자동으로 테스트 관련 라이브러리(pytest, JUnit 등)를 불러오고 테스트 코드를 제안할 준비를 합니다.
* **파일명의 힌트:** `auth_service.py`라는 파일명만 보고도 AI는 이 파일이 인증과 관련된 로직을 담고 있을 것이라고 판단하며, 관련 스키마나 보안 프로토콜을 우선적으로 참고합니다.
* **설정 파일 참조:** 루트 폴더에 있는 `pyproject.toml`이나 `package.json`을 읽어 현재 프로젝트가 사용하는 라이브러리 버전과 의존성을 파악합니다.

### 2.2. 표준 구조가 제안 정확도를 높이는 이유
* **검색 범위의 최적화:** 구조가 표준화되어 있으면 Copilot이 '주변 탭(Neighboring Tabs)'이나 프로젝트 내 관련 코드를 찾을 때 훨씬 정확한 파일을 선택할 수 있습니다.
* **패턴 매칭의 효율성:** AI는 수억 개의 오픈소스 프로젝트를 학습했습니다. 대부분의 오픈소스가 표준 구조를 따르기 때문에, 사용자의 프로젝트가 표준을 따를수록 학습 데이터와의 일치율이 높아져 더 정교한 코드를 제안하게 됩니다.

> **핵심 포인트:** 비표준적인 구조(예: 모든 파일을 한 폴더에 넣기)는 AI에게 "눈을 감고 코드를 짜라"고 요구하는 것과 같습니다. 구조를 표준화하는 것은 AI에게 가장 강력한 안경을 씌워주는 것과 같습니다.

## 3. 실습 가이드: 구조에 따른 제안 차이 확인

1. **비표준 구조 테스트:** 아무 폴더 안에 `a.py` 파일을 만들고 "로그인 함수 작성"을 요청해봅니다.
2. **표준 구조 테스트:** `src/services/auth_service.py`와 같이 경로와 이름을 명확히 지정한 후 동일한 요청을 해봅니다.
3. **결과 비교:** 파일명과 경로만으로도 Copilot이 얼마나 더 전문적이고 맥락에 맞는 변수명과 로직을 제안하는지 확인합니다.
