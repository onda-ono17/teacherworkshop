## 1. Phase 1: 기본 프로젝트 구조 (Scaffolding Hint)

@workspace /new 명령어를 통해 생성된 초기 main.py의 표준적인 기초 코드입니다.

**파일 위치: src/main.py**

from fastapi import FastAPI
from pydantic import BaseModel
from typing import List, Optional

app = FastAPI()

# 도서 데이터 모델 정의
class Book(BaseModel):
    id: int
    title: str
    author: str
    description: Optional[str] = None

# 메모리 데이터베이스 역할을 할 리스트
books_db: List[Book] = []

@app.get("/")
def read_root():
    return {"message": "Welcome to Book Management API"}

## 2. Phase 2: 검색 로직 및 리팩토링 (Refactoring Hint)

수강생들이 리팩토링하기 전의 기초적인 검색 로직 예시입니다. 이 코드를 작성한 후 인라인 채팅(Ctrl+I / Cmd+I)을 사용하여 최적화하도록 유도하십시오.

**수정 전 코드 (비효율적 반복문):**

@app.get("/books/search")
def search_books(keyword: str):
    # 인라인 채팅으로 리팩토링할 대상 코드 블록
    found_books = []
    for book in books_db:
        if keyword.lower() in book.title.lower():
            found_books.append(book)
    return found_books

**인라인 채팅 지시어 예시:**
"이 검색 로직을 리스트 컴프리헨션을 사용하여 한 줄로 변경하고, 결과가 없을 경우 빈 리스트 대신 적절한 메시지를 반환하도록 수정해줘."

## 3. Phase 3: 단위 테스트 구성 (Testing Hint)

/tests 명령어를 사용하여 생성된 pytest 코드의 기본 구조입니다. 수강생들이 테스트 파일의 위치와 형식을 익히는 데 활용합니다.

**파일 위치: tests/test_main.py**

import pytest
from fastapi.testclient import TestClient
from src.main import app

client = TestClient(app)

def test_read_main():
    response = client.get("/")
    assert response.status_code == 200
    assert response.json() == {"message": "Welcome to Book Management API"}

def test_create_book():
    # 도서 등록 기능을 테스트하는 시나리오
    book_data = {"id": 1, "title": "GitHub Copilot Guide", "author": "Ideacube"}
    response = client.post("/books/", json=book_data)
    assert response.status_code == 200
