# 실습: GitHub Copilot과 함께하는 API 서버 구축 챌린지

본 실습은 "기획 -> 뼈대 생성 -> 핵심 로직 구현 -> 품질 확보 -> 실행 및 테스트"의 현대적 개발 사이클을 GitHub Copilot과 협업하여 단시간 내에 완수하는 것을 목표로 합니다.

---

## 1. 실습 목표
* @workspace /new를 이용한 표준 프로젝트 구조 생성
* 인라인 채팅을 통한 비즈니스 로직 구현 및 리팩토링
* /tests 및 /doc을 활용한 코드 신뢰도 및 가독성 확보
* /terminal을 통한 로컬 환경 실행 및 관리

---

## 2. 실습 단계별 미션

### Phase 1: 프로젝트 뼈대 구축 (Scaffolding)
**미션:** FastAPI와 Poetry를 사용하는 도서 관리 시스템의 기본 구조를 생성하세요.
* **방법:** 측면 채팅에서 @workspace /new 명령어를 사용합니다.
* **프롬프트 예시:** @workspace /new Python 기반의 도서 관리 API 서버를 만들어줘. FastAPI와 Poetry를 사용하고, 도서 데이터는 메모리에 리스트 형태로 저장해. 구조는 src/ 폴더 아래에 기능별로 나눠줘.
* **결과 확인:** pyproject.toml, main.py, 그리고 표준 폴더 구조가 생성되었는지 확인하고 [Create Workspace]를 누릅니다.

---

### Phase 2: 핵심 로직 구현 및 리팩토링 (Inline Chat)
**미션:** 특정 조건으로 도서를 검색하는 기능을 추가하고 코드를 깔끔하게 다듬으세요.
* **작업 1:** 도서 제목(Title)의 일부만 입력해도 검색되는 기능을 추가합니다.
* **작업 2:** 인라인 채팅(Ctrl+I / Cmd+I)을 사용하여 "이 검색 로직을 리스트 컴프리헨션으로 변경하고 가독성 좋게 리팩토링해줘"라고 요청합니다.
* **작업 3:** 결과 확인 후 Accept를 눌러 코드를 반영합니다.

---

### Phase 3: 품질 확보 및 문서화 (/tests, /doc)
**미션:** 완성된 API의 신뢰성을 높이고 표준화된 문서화 주석을 작성하세요.
* **작업 1:** 주요 CRUD 함수를 선택하고 /doc 명령어를 사용하여 Google 스타일의 Docstring을 추가합니다.
* **작업 2:** /tests 명령어를 사용하여 pytest 기반의 단위 테스트 코드를 생성합니다. 특히 검색 결과가 없을 때의 예외 처리가 포함되는지 확인합니다.
* **작업 3:** 생성된 테스트 코드를 test_main.py로 저장합니다.

---

### Phase 4: 터미널 제어 및 실행 (/terminal)
**미션:** 브라우저 검색 없이 터미널 명령어만으로 서버를 실행하고 테스트하세요.
* **방법:** /terminal 명령어를 사용하여 "FastAPI 서버를 uvicorn으로 실행하는 명령어를 알려줘"라고 요청합니다.
* **작업 1:** 제안된 명령어를 [Insert into Terminal] 버튼으로 입력하여 서버를 구동합니다.
* **작업 2:** 새로운 터미널 탭에서 /terminal에게 "curl 명령어로 새 도서를 등록하는 예시를 작성해줘"라고 요청하여 실제 API 동작을 확인합니다.

---

## 3. 최종 제출 및 검토 가이드

수강생은 최종적으로 다음과 같은 상태의 프로젝트를 완성해야 합니다.
1. 표준 구조: src/, tests/ 폴더가 구분된 전문적인 폴더 구조.
2. 리팩토링된 코드: 리스트 컴프리헨션 등을 활용한 간결하고 파이썬다운 로직.
3. 높은 가독성: 모든 주요 함수에 타입 힌트와 표준 Docstring 적용.
4. 검증된 코드: 생성된 단위 테스트를 pytest로 실행하여 모두 통과한 상태.

---

## 4. 학습 포인트 정리
* AI는 단순히 코드를 작성하는 도구를 넘어, 전체 개발 수명 주기를 보조하는 엔지니어링 파트너임을 체감합니다.
* 좋은 프로젝트 구조와 명확한 파일명이 AI의 제안 품질을 결정한다는 점을 이해합니다.
* 생성된 코드를 비판적으로 검토하고 테스트를 통해 검증하는 습관을 기릅니다.
