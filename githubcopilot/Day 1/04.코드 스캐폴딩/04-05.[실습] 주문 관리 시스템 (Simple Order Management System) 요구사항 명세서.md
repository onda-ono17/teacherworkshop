# 실습: 주문 관리 시스템(Simple Order Management System) 요구사항 명세서

본 문서는 수강생이 자연어로 작성된 비즈니스 요구사항을 이해하고, GitHub Copilot을 활용하여 실제 구동 가능한 코드로 구현하는 연습을 위해 작성되었습니다.

---

## 1. 프로젝트 개요
* 프로젝트명: 간단한 주문 관리 시스템 (Simple Order Management System)
* 목적: 회원, 상품, 주문 간의 관계를 이해하고 비즈니스 로직(재고 확인, 금액 계산)을 구현함
* 기술 스택 (택 1):
    - Python: FastAPI + SQLAlchemy + Pydantic
    - Java: Spring Boot + Spring Data JPA + H2 Database

---

## 2. 도메인 모델 및 엔티티 요구사항

### 2.1. 회원 (Member)
* 속성: 회원 ID(PK), 이름, 이메일, 가입 날짜
* 제약: 이메일은 중복될 수 없음

### 2.2. 상품 (Product)
* 속성: 상품 ID(PK), 상품명, 가격, 재고 수량
* 제약: 가격과 재고는 0 이상이어야 함

### 2.3. 주문 및 주문 항목 (Order & OrderItem)
* 주문 속성: 주문 ID(PK), 회원 ID(FK), 주문 상태, 총 결제 금액, 주문 일시
* 주문 항목 속성: 주문 항목 ID(PK), 주문 ID(FK), 상품 ID(FK), 주문 수량, 주문 당시 가격

---

## 3. 주요 기능 요구사항 상세

### 3.1. 회원 및 상품 관리
* 새로운 회원을 등록하고 정보를 조회할 수 있어야 함
* 상품의 이름, 가격, 초기 재고를 등록할 수 있어야 함

### 3.2. 주문 처리 로직 (핵심)
* 회원은 한 번의 주문으로 여러 종류의 상품을 각각 다른 수량으로 주문할 수 있음
* [재고 검증]: 주문 시 각 상품의 현재 재고를 확인하며, 하나라도 재고가 부족할 경우 주문 전체를 실패 처리하고 적절한 오류 메시지를 반환함
* [재고 차감]: 주문이 성공하면 각 상품의 재고 수량을 주문 수량만큼 감소시킴
* [금액 계산]: 주문된 모든 상품의 (수량 * 가격)을 합산하여 총 결제 금액을 자동 계산하고 저장함

### 3.3. API 설계 원칙
* RESTful API 원칙을 준수함 (예: POST /orders, GET /members/{id})
* 적절한 HTTP 상태 코드(200 OK, 201 Created, 400 Bad Request 등)를 반환함

---

## 4. Copilot 활용 실습 시나리오 (수강생 가이드)

수강생은 아래 단계에 따라 Copilot에게 지시하며 구현을 진행합니다.

### 단계 1: 프로젝트 구조 및 스캐폴딩
* 프롬프트: "@workspace /new FastAPI를 사용한 주문 관리 시스템 프로젝트를 만들어줘. SQLAlchemy를 사용하고 회원, 상품, 주문 엔티티가 포함되어야 해."

### 단계 2: 엔티티 및 스키마 정의
* 프롬프트: "SQLAlchemy를 사용하여 Member, Product, Order, OrderItem 모델을 정의해줘. 주문과 주문 항목은 1:N 관계야."

### 단계 3: 비즈니스 로직 구현 (주문 처리)
* 프롬프트: "주문 생성 로직을 작성해줘. 주문 시 상품 재고를 확인해서 부족하면 Exception을 발생시키고, 성공하면 재고를 줄인 뒤 총 금액을 계산해서 저장해야 해. 이 모든 과정은 하나의 트랜잭션으로 처리해줘."

### 단계 4: 단위 테스트 생성
* 프롬프트: "/tests 주문 성공 시나리오와 재고 부족으로 인한 주문 실패 시나리오에 대한 테스트 코드를 작성해줘."
