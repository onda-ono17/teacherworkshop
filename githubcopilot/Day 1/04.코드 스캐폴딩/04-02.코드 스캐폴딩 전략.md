# 효율적인 코드 스캐폴딩(Scaffolding) 전략

본 세션에서는 새로운 프로젝트를 시작할 때 수동으로 폴더와 파일을 생성하는 대신, GitHub Copilot을 활용하여 **Top-down(하향식)** 방식으로 프로젝트의 뼈대를 순식간에 구축하는 전략을 학습합니다.

---

## 1. 하향식(Top-down) 접근 전략

성공적인 프로젝트 구축의 핵심은 '큰 그림'에서 '세부 구현'으로 내려가는 것입니다. Copilot은 이 과정을 단계별로 자동화해 줍니다.

### [Step 1] 프로젝트 전체 구조 제안받기
* **방법:** 측면 채팅에서 만들고자 하는 서비스의 성격(예: FastAPI 기반의 주문 관리 시스템)을 설명하고 전체 폴더 구조를 요청합니다.
* **효과:** 표준화된 디렉터리 구조(`src`, `app`, `tests` 등)를 즉시 확보하여 초기 설계 시간을 단축합니다.

### [Step 2] 의존성 및 설정 파일 생성
* **방법:** 제안받은 구조에 맞춰 `pyproject.toml`(Python), `pom.xml`(Java), 또는 `package.json`(Node.js) 생성을 요청합니다.
* **효과:** 필요한 라이브러리 조합과 버전을 AI가 추천해 주어 의존성 충돌을 사전에 방지합니다.
* **⚠️주의사항(Version Control):** AI의 학습 데이터 시점으로 인해 간혹 과거 버전의 라이브러리(예: Pydantic V1 등)를 제안할 수 있습니다. 프롬프트에 *"가장 최신 버전 호환성에 맞게 작성해줘"* 또는 *"의존성에 특정 버전(예: Spring Boot 3.2.x)을 사용해줘"* 라고 명시하는 것이 중요합니다.

### [Step 3] 핵심 엔티티(Entity) 및 스키마 정의
* **방법:** 데이터베이스 모델이나 API 스키마를 정의합니다.
* **프롬프트:** "주문 시스템을 위한 `Order`, `Product`, `User` 엔티티를 SQLAlchemy 기반으로 작성해줘."

### [Step 4] 보일러플레이트(Boilerplate) 코드 생성
* **방법:** 반복적으로 발생하는 CRUD(Create, Read, Update, Delete) 로직이나 에러 핸들링 코드를 생성합니다.

## 2. @workspace /new 명령어 활용하기

GitHub Copilot Chat의 가장 강력한 명령어 중 하나인 **`/new`**는 설명한 하향식 전략을 단 하나의 명령어로 통합 실행합니다.

### 2.1. 명령어 사용법
측면 채팅창에 다음과 같이 입력합니다:
> **@workspace /new** Python 기반의 네트워크 자동화 관리 도구 프로젝트를 만들어줘. Poetry를 사용하고, API는 FastAPI를 사용해.

### 2.2. 작동 방식
1.  **구조 설계:** AI가 프로젝트에 필요한 모든 디렉터리와 파일 목록을 생성합니다.
2.  **파일 내용 생성:** 각 파일에 들어갈 초기 기초 코드를 작성합니다.
3.  **미리보기:** 제안된 구조를 확인한 후 **[Create Workspace]** 버튼을 누르면 실제 로컬 경로에 파일들이 생성됩니다.



---

## 3. 스캐폴딩의 정교함 높이기 (Tips)

* **프레임워크 및 프레임워크 버전 명시:** "FastAPI 최신 버전 사용해줘"와 같이 구체적인 기술 스택과 버전을 언급할수록 보일러플레이트와 의존성의 호환성 문제가 크게 줄어듭니다.
* **설계 원칙 적용:** "Clean Architecture 구조로 짜줘" 또는 "DDD(Domain Driven Design) 패턴을 적용해줘"라고 요청하여 수준 높은 아키텍처를 확보할 수 있습니다.
* **연속 수정:** `/new`로 만들어진 초기 구조가 맘에 들지 않는다면, 채팅창에서 "여기에 Dockerfile도 추가해줘"라고 연속해서 요청하십시오.

---

## 4. 실습: 5분 만에 프로젝트 뼈대 만들기

1.  VS Code에서 Copilot Chat을 엽니다.
2.  `@workspace /new` 명령어를 사용하여 본인이 평소 관심 있던 작은 토이 프로젝트(예: 개인 블로그 시스템, 주식 알림 봇 등)의 구조를 요청합니다.
3.  제안된 `pyproject.toml` 또는 `requirements.txt`에 필요한 라이브러리들이 잘 포함되었는지 확인합니다.
4.  **[Create Workspace]**를 눌러 실제 폴더가 생성되는 것을 확인하고, 각 파일의 초기 코드를 검토합니다.
